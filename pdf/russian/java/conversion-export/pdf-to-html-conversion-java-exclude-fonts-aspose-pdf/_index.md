# Как преобразовать PDF в HTML в Java с помощью Aspose.PDF: исключить определенные шрифты

## Введение

Конвертация PDF в HTML с одновременным контролем использования шрифтов может быть сложной задачей. В этом руководстве показано, как использовать библиотеку Aspose.PDF для Java, чтобы добиться этого, гарантируя единообразный вид ваших документов на веб-платформах.

**Что вы узнаете:**
- Как исключить определенные шрифты при конвертации PDF в HTML с помощью Aspose.PDF для Java.
- Методы точной настройки вывода с дополнительными параметрами конфигурации.
- Лучшие практики и реальные приложения для оптимизации производительности.

Начнем с настройки среды разработки.

## Предпосылки

Перед началом убедитесь, что у вас есть:

### Требуемые библиотеки, версии и зависимости

Вам понадобится Aspose.PDF для Java версии 25.3 или более поздней.

### Требования к настройке среды

- Установлен совместимый комплект разработки Java (JDK).
- Интегрированная среда разработки (IDE), например IntelliJ IDEA, Eclipse или NetBeans для разработки и тестирования.

### Необходимые знания

Базовые знания программирования на Java и работы с файлами будут преимуществом.

## Настройка Aspose.PDF для Java

Чтобы использовать Aspose.PDF для Java, включите его в свой проект через Maven или Gradle:

**Мейвен:**

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-pdf</artifactId>
    <version>25.3</version>
</dependency>
```

**Градл:**

```gradle
implementation 'com.aspose:aspose-pdf:25.3'
```

### Приобретение лицензии

Aspose.PDF для Java требует лицензию. Вы можете начать с бесплатной пробной версии или запросить временную лицензию для расширенного тестирования.

#### Базовая инициализация и настройка

После добавления Aspose.PDF в ваш проект инициализируйте его следующим образом:

```java
import com.aspose.pdf.Document;
```

Обязательно настройте пути к каталогам для входных PDF-файлов и выходных HTML-файлов.

## Руководство по внедрению

В нашем руководстве представлены основные параметры исключения шрифтов и расширенные параметры конфигурации.

### Функция 1: Базовое исключение шрифтов при конвертации PDF в HTML

Эта функция позволяет преобразовать PDF-документ в HTML, исключив определенные шрифты, что гарантирует единообразный вид веб-страниц без лишних ресурсов шрифтов.

#### Обзор

Aspose.PDF по умолчанию копирует стиль оригинального PDF. Вы можете исключить определенные шрифты для лучшего контроля над выводом.

#### Этапы внедрения

**Шаг 1: Настройте пути к файлам**

Определите каталоги и пути к файлам:

```java
String dataDir = "YOUR_DOCUMENT_DIRECTORY";
String outputDir = "YOUR_OUTPUT_DIRECTORY";
```

**Шаг 2: Инициализация `HtmlSaveOptions` с настройками исключения шрифтов**

Настройте параметры сохранения HTML:

```java
HtmlSaveOptions htmlOptions = new HtmlSaveOptions();
htmlOptions.setExcludeFontNameList(new String[]{"Arial", "Calibri"});
htmlOptions.setDefaultFontName("Arial Black");
```

**Шаг 3: Загрузите и сохраните PDF-документ**

Загрузите ваш PDF-документ и примените параметры сохранения:

```java
Document pdfDocument = new Document(dataDir + "/ExcludeFont.pdf");
pdfDocument.save(outputDir + "/ExcludeFontResources.html", htmlOptions);
```

### Функция 2: Расширенная настройка исключения шрифтов

Улучшите контроль над выводом HTML с помощью дополнительных параметров конфигурации.

#### Обзор

Расширенные настройки позволяют выполнять точные настройки, включая согласованность макета и обработку изображений. Вот как использовать эти функции:

#### Этапы внедрения

**Шаг 1: Настройте дополнительные `HtmlSaveOptions`**

Настройте параметры сохранения с помощью дополнительных параметров:

```java
HtmlSaveOptions htmlOptions = new HtmlSaveOptions();
htmlOptions.setExplicitListOfSavedPages(new int[]{1});
htmlOptions.setFixedLayout(true);
htmlOptions.setCompressSvgGraphicsIfAny(false);
htmlOptions.setSaveTransparentTexts(true);
htmlOptions.setSaveShadowedTextsAsTransparentTexts(true);

htmlOptions.setExcludeFontNameList(new String[]{"ArialMT", "SymbolMT"});
htmlOptions.setDefaultFontName("Comic Sans MS");

htmlOptions.setUseZOrder(true);
htmlOptions.setLettersPositioningMethod(LettersPositioningMethods.UseEmUnitsAndCompensationOfRoundingErrorsInCss);
htmlOptions.setPartsEmbeddingMode(HtmlSaveOptions.PartsEmbeddingModes.NoEmbedding);

htmlOptions.setRasterImagesSavingMode(HtmlSaveOptions.RasterImagesSavingModes.AsEmbeddedPartsOfPngPageBackground);
htmlOptions.setSplitIntoPages(false);
```

**Шаг 2: Загрузка и сохранение с расширенными параметрами**

```java
Document pdfDocument = new Document(dataDir + "/ExcludeFontResourcesWithAdditionalOptions.pdf");
pdfDocument.save(outputDir + "/ExcludeFontResourcesWithAdditionalOptions.html", htmlOptions);
```

### Советы по устранению неполадок

- **Шрифты не исключены**: Убедитесь, что названия шрифтов точно соответствуют названиям в PDF-файле.
- **Проблемы с макетом**: Проверять `HtmlSaveOptions` настройки свойств макета, такие как `setFixedLayout`.
- **Использование памяти**: Контролируйте использование памяти и при необходимости корректируйте настройки JVM для больших документов.

## Раздел часто задаваемых вопросов

**1. Как обращаться со шрифтами, не указанными в `setExcludeFontNameList`?**
Убедитесь, что все названия шрифтов указаны точно так, как они указаны в PDF-файле, с учетом регистра.

**2. Могу ли я использовать этот подход для пакетной обработки нескольких документов?**
Да, пройдитесь по коллекции файлов и примените эти настройки к каждому документу по отдельности.

**3. Что делать, если я хочу встроить шрифты, а не исключать их?**
Отрегулируйте свой `HtmlSaveOptions` удалив или закомментировав `setExcludeFontNameList` вызов метода.

**4. Существуют ли какие-либо ограничения по использованию Aspose.PDF для Java?**
Несмотря на свою эффективность, для полной функциональности после окончания пробного периода требуется действующая лицензия.

**5. Как мне получить поддержку в случае необходимости?**
Для получения дополнительной помощи обратитесь к документации Aspose или в службу поддержки.
