# Преобразование PDF в HTML: Захват предупреждений о замене шрифтов с помощью Aspose.PDF для Java

## Введение

Когда вы выполняете **pdf to html conversion**, замена шрифтов может незаметно изменить внешний вид ваших страниц, вызывая сдвиги макета или отсутствие символов. Захват этих предупреждений позволяет убедиться, что преобразование сохраняет оригинальный дизайн, и помогает обнаружить отсутствующие шрифты pdf до того, как они станут проблемой. В этом руководстве вы узнаете, как подключиться к конвейеру преобразования Aspose.PDF для Java, записывать любые изменения шрифтов и сохранять полученный HTML‑файл с уверенностью.

**Что вы достигнете:**
- Понять, почему мониторинг замены шрифтов важен при pdf to html conversion.
- Настроить обработчик замены шрифтов, который фиксирует каждое изменение шрифта.
- Настроить `HtmlSaveOptions` для точной настройки результата преобразования.

Убедимся, что у вас есть всё необходимое, прежде чем мы начнём.

## Быстрые ответы
- **Что делает обработчик замены шрифтов?** Он записывает оригинальное название шрифта и шрифт, который Aspose.PDF заменяет во время преобразования.  
- **Можно ли использовать это в проектах pdf to html java?** Да, код работает с любым Java‑приложением, которое использует Aspose.PDF.  
- **Нужна ли лицензия для продакшн‑использования?** Для коммерческих развертываний требуется действительная лицензия Aspose.PDF.  
- **Будут ли автоматически обнаруживаться отсутствующие шрифты?** Обработчик записывает каждую замену, эффективно позволяя вам обнаруживать отсутствующие шрифты pdf.  
- **Требуется ли дополнительная конфигурация?** Только стандартная настройка Aspose.PDF и регистрация обработчика, показанные ниже.

## Что такое pdf to html conversion?
Pdf to html conversion преобразует PDF‑документ в веб‑дружелюбный HTML‑файл, пытаясь сохранить оригинальный макет, шрифты и изображения. Этот процесс полезен для отображения PDF‑файлов в браузерах без необходимости в плагине PDF‑просмотрщика.

## Почему стоит захватывать предупреждения о замене шрифтов?
Во время преобразования, если оригинальный шрифт не встроен или недоступен в системе, Aspose.PDF заменяет его запасным. Без видимости HTML может выглядеть заметно иначе. Захватывая предупреждения, вы можете:
- Выявить отсутствующие шрифты на ранней стадии.
- Выбрать встраивание необходимых шрифтов.
- Предоставить стратегию запасных вариантов для конечных пользователей.

## Руководство по реализации

### Функция: Предупреждение о замене шрифтов в pdf to html conversion

Эта функция позволяет отслеживать и фиксировать любые замены шрифтов, происходящие при преобразовании PDF в HTML.

#### Шаг 1: Загрузите ваш PDF‑документ
(Уже показано выше) Загрузка документа дает доступ к его содержимому и информации о шрифтах.

#### Шаг 2: Настройте обработчик замены шрифтов
Зарегистрируйте обработчик, который записывает каждую замену в карту для последующего анализа.

```java
final Map<String, String> names = new HashMap<>();
pdfDoc.FontSubstitution.add(new Document.FontSubstitutionHandler() {
    public void invoke(Font font, Font newFont) {
        // Log substituted FontNames into a map.
        names.put(font.getFontName(), newFont.getFontName());
    }
});
```

**Почему это важно:**  
Если при преобразовании проприетарный шрифт заменяется на общий, HTML может отобразиться с неожиданными пробелами или отсутствующими глифами. Карта `names` предоставляет чёткую трассировку изменений.

#### Шаг 3: Настройте параметры сохранения HTML
Создайте экземпляр `HtmlSaveOptions` для управления тем, как PDF сохраняется в виде HTML.

```java
HtmlSaveOptions htmlSaveOps = new HtmlSaveOptions();
```

Вы можете дополнительно настроить свойства, такие как `SplitIntoPages`, `EmbedFonts` или `ImageCompression`, в зависимости от потребностей вашего проекта.

#### Шаг 4: Сохраните преобразованный документ
Наконец, запишите HTML‑вывод на диск.

```java
pdfDoc.save("YOUR_OUTPUT_DIRECTORY/getWarningForFontSubstitution.html\
```

После выполнения проверьте карту `names`, чтобы увидеть, какие шрифты были заменены. Если вы заметите неожиданные записи, рассмотрите возможность встраивания недостающих шрифтов или корректировки настроек преобразования.

## Распространённые проблемы и их устранение

| Симптом | Вероятная причина | Решение |
|---------|-------------------|---------|
| Отсутствуют записи в карте `names` | Замена шрифтов отключена или все шрифты встроены | Убедитесь, что `EmbedFonts` установлен в `false` в `HtmlSaveOptions`, если хотите видеть замены. |
| Разметка HTML нарушена | Заменённый шрифт не содержит необходимых глифов | Встроите недостающий шрифт или предоставьте CSS‑запасной вариант, соответствующий оригинальному дизайну. |
| `pdfDoc.save` бросает исключение | Неправильный путь вывода или отсутствие прав записи | Убедитесь, что каталог `YOUR_OUTPUT_DIRECTORY` существует и доступен для записи. |

## Часто задаваемые вопросы

**В: Можно ли использовать этот подход с другими форматами вывода (например, DOCX)?**  
**О:** Да. Aspose.PDF предоставляет аналогичные события замены шрифтов для большинства целей преобразования.

**В: Как обнаружить отсутствующие шрифты pdf до преобразования?**  
**О:** Проверьте коллекцию `pdfDoc.FontInfo` или полагайтесь на обработчик замены шрифтов во время преобразования.

**В: Есть ли способ автоматически встраивать недостающие шрифты?**  
**О:** Установите `htmlSaveOps.setEmbedFonts(true)`; Aspose.PDF встроит доступные шрифты, но действительно отсутствующие шрифты необходимо предоставить вручную.

**В: Работает ли это с зашифрованными PDF?**  
**О:** Да, при условии, что вы предоставите пароль при загрузке документа: `new Document(path, new LoadOptions(password))`.

**В: Увеличит ли это время преобразования?**  
**О:** Накладные расходы на запись замен минимальны, обычно добавляют лишь несколько миллисекунд.
