## Настройка Aspose.PDF для Java

### Установка через Maven

Чтобы добавить Aspose.PDF в ваш проект с помощью Maven, включите следующую зависимость в ваш `pom.xml`:

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-pdf</artifactId>
    <version>25.3</version>
</dependency>
```

### Установка через Gradle

Для проектов, использующих Gradle, добавьте следующую строку в свой файл `build.gradle`:

```gradle
implementation 'com.aspose:aspose-pdf:25.3'
```

## Руководство по внедрению

### Обзор функций: преобразование PDF в HTML с заменой шрифтов

Эта функция позволяет конвертировать PDF-документы в формат HTML, заменяя недоступные шрифты указанными альтернативами, обеспечивая визуальную согласованность.

#### Шаг 1: Настройте каталоги и загрузите документ

Начните с указания путей для входного PDF-файла и выходного HTML-файла:

```java
String dataDir = "YOUR_DOCUMENT_DIRECTORY/input1.pdf"; // Входной путь PDF
String outputDir = "YOUR_OUTPUT_DIRECTORY/pdfToHTMLWithFontSubstitution.html"; // Выходной HTML-путь

// Загрузить документ из указанного каталога
Document pdf = new Document(dataDir);
```

#### Шаг 2: Настройка логики замены шрифтов

Создайте пользовательский подкласс `CustomFontSubstitutionBase` для обработки замены шрифтов:

```java
CustomSubst1 subst1 = new CustomSubst1();
FontRepository.getSubstitutions().add(subst1);
```

#### Шаг 3: Добавьте уведомитель для событий замены шрифтов

Зарегистрируйте обработчик событий для регистрации или вывода сведений о замене шрифта:

```java
pdf.FontSubstitution.add(new Document.FontSubstitutionHandler() {
    public void invoke(Font font, Font newFont) {
        System.out.println("Original Font: " + font.getFontName() + ", New Font: " + newFont.getFontName());
    }
});
```

#### Шаг 4: Преобразование PDF в HTML

Настройте `HtmlSaveOptions` и сохраните документ как HTML-файл:

```java
HtmlSaveOptions options = new HtmlSaveOptions();
pdf.save(outputDir, options);
```
