### Разделение страниц PDF

Эта функция позволяет разделить один PDF-документ на отдельные файлы, каждый из которых содержит одну страницу. 

#### Шаг 1: Инициализация PdfFileEditor

Сначала создайте экземпляр `PdfFileEditor`, который предоставляет методы для работы с PDF-файлами:

```csharp
using Aspose.Pdf.Facades;

PdfFileEditor pdfEditor = new PdfFileEditor();
```

**Почему**: Этот класс имеет решающее значение для доступа к `SplitToPages` метод, использованный позже.

#### Шаг 2: Задайте пути к каталогам

Укажите пути к входным и выходным каталогам для эффективного управления расположением файлов:

```csharp
string dataDir = "YOUR_DOCUMENT_DIRECTORY";
string outputDir = "YOUR_OUTPUT_DIRECTORY";
```

**Почему**: Организация файлов помогает поддерживать ясность, особенно в крупных проектах.

#### Шаг 3: Откройте PDF-файл как поток

Откройте входной PDF-файл с помощью `FileStream` чтобы обработать его как поток:

```csharp
using System.IO;

FileStream inputStream = new FileStream(dataDir + "/input.pdf", FileMode.Open);
```

**Почему**: Потоки эффективно используют память для обработки больших файлов.

#### Шаг 4: Разделите PDF-файл на страницы

Использовать `SplitToPages`, который возвращает массив `MemoryStream` объекты, каждый из которых представляет страницу:

```csharp
MemoryStream[] outBuffer = pdfEditor.SplitToPages(inputStream);
```

**Почему**: Этот метод эффективно разбивает страницы без создания промежуточных файлов.

#### Шаг 5: Сохраните каждую страницу по отдельности

Просмотрите полученные потоки и сохраните их как отдельные PDF-файлы, используя `FileStream`:

```csharp
int fileNumber = 1;
foreach (MemoryStream aStream in outBuffer)
{
    using (FileStream outStream = new FileStream(outputDir + "/File_" + fileNumber.ToString() + "_out.pdf", FileMode.Create))
    {
        aStream.WriteTo(outStream);
    }
    fileNumber++;
}
```

**Почему**: Этот шаг гарантирует, что каждая страница будет сохранена как отдельный документ, что упрощает управление.

#### Шаг 6: Очистка

Всегда закрывайте входной поток после обработки:

```csharp
inputStream.Close();
```

**Почему**: Правильное закрытие потоков предотвращает утечки памяти и блокировку ресурсов.
