# Создание PDF‑документа C# – Полное руководство с тегированным текстом и позиционированием

Когда‑нибудь задумывались, как **создать PDF‑документ C#**, который выглядит привлекательно и при этом имеет правильную структуру для доступности? Вы не одиноки. Многие разработчики сталкиваются с проблемой, когда нужно добавить пустую страницу PDF, внедрить тегированный контент и точно разместить текст — всё в одном процессе.

В этом руководстве мы пройдём через полностью готовый к запуску пример, показывающий **как создать тегированный PDF** с помощью Aspose.Pdf для .NET. К концу вы получите PDF с пустой страницей, элементом `<span>`, размещённым в точке (100, 200), и сохранённым как тегированный PDF, готовый для скрин‑ридеров.

## Что вы узнаете

- Как **создать PDF‑документ C#** с нуля с помощью Aspose.Pdf.  
- Самый простой способ **добавить пустую страницу PDF** в ваш документ.  
- Точные шаги **как создать тегированный PDF** с использованием API TaggedContent.  
- Как **установить позицию текста PDF** с пиксель‑точными координатами.  
- Советы, подводные камни и идеи для дальнейшего расширения примера.

### Предварительные требования

- .NET 6.0 или новее (код также работает на .NET Framework 4.6+).  
- Действительная лицензия Aspose.Pdf для .NET или бесплатный оценочный ключ.  
- Visual Studio 2022 или любой другой предпочитаемый редактор C#.

Дополнительные пакеты NuGet не требуются, кроме `Aspose.Pdf`.

## Создание PDF‑документа C# – Инициализация Aspose.Pdf

Прежде всего, чтобы **создать PDF‑документ C#**, нам нужен экземпляр `Aspose.Pdf.Document`. Представьте этот объект как пустой холст, на котором будет размещено всё остальное.

```csharp
using Aspose.Pdf;
using Aspose.Pdf.Text;

// Step 1: Initialize a new PDF document
using (var doc = new Document())
{
    // The Document constructor creates a blank PDF ready for pages and content.
```

> **Полезный совет:** Оберните `Document` в блок `using`. Это гарантирует освобождение файлового дескриптора и предотвращает блокировку файла в Windows.

## Добавление пустой страницы PDF с помощью Aspose

Теперь, когда у нас есть документ, мы **добавим пустую страницу PDF**. PDF без страниц — это по сути оболочка, бесполезная в большинстве сценариев.

```csharp
    // Step 2: Add a blank page to the document
    var page = doc.Pages.Add();   // Returns a freshly created Page object
```

Метод `Pages.Add()` добавляет новую страницу в конец коллекции. Если нужна конкретная размерность, можно передать перечисление `PageSize` или указать пользовательские размеры:

```csharp
    // Example: Add an A4‑sized page (optional)
    // var page = doc.Pages.Add(PageSize.A4);
```

> **Почему это важно:** Добавление пустой страницы даёт чистую поверхность для последующего размещения тегированных элементов, изображений или таблиц.

## Как создать тегированный PDF с элементами Span

Тегированные PDF‑файлы критически важны для доступности; они позволяют вспомогательным технологиям понимать логическую структуру. Aspose.Pdf предоставляет дерево `TaggedContent`, в которое можно вставлять такие элементы, как `Span`.

```csharp
    // Step 3: Create a span element for tagged content
    var span = doc.TaggedContent.CreateSpanElement();
```

`Span` представляет собой последовательность текста. По умолчанию он наследует стиль окружающего контекста, но вы можете задать собственные шрифты, цвета и т.д.

```csharp
    // Optional: Change the font style (demonstrates flexibility)
    span.Font = FontRepository.FindFont("Arial");
    span.FontSize = 12;
```

## Точное позиционирование текста в PDF

Следующая задача — **установить позицию текста PDF**. Мы хотим, чтобы наш `<span>` появился в координатах (100, 200), измеряемых от нижнего левого угла страницы.

```csharp
    // Step 4: Define the displayed text and its exact position
    span.Text = "Tagged text at (100,200)";
    span.Position = new Position(100, 200); // X = 100, Y = 200
```

Зачем использовать `Position` вместо более высокого уровня компоновки? Потому что для тегированных PDF часто требуется абсолютное размещение, например, при наложении текста на отсканированную форму.

> **Распространённый вопрос:** *Что делать, если нужно разместить текст относительно правого верхнего угла?*  
> Просто вычислите координату Y как `page.PageInfo.Height - желаемыйСмещение` и задайте `Position` соответствующим образом.

## Добавление Span в дерево Tagged Content

Когда `<span>` готов, мы присоединяем его к корню дерева тегированного контента. Этот шаг фактически регистрирует элемент в логической структуре PDF.

```csharp
    // Step 5: Append the span to the root element of the tagged content tree
    doc.TaggedContent.RootElement.AppendChild(span);
```

Если вы строите более сложную иерархию (разделы, абзацы, таблицы), создайте промежуточные узлы, такие как `Div` или `Paragraph`, и присоедините к ним `<span>`.

## Сохранение и проверка тегированного PDF

Наконец, сохраняем документ на диск. Файл будет содержать пустую страницу, тегированный `<span>` и правильную структуру.

```csharp
    // Step 6: Save the PDF with the tagged text
    doc.Save("tagged_output.pdf");
}
```

### Ожидаемый результат

Откройте `tagged_output.pdf` в Adobe Acrobat Reader:

- Вы увидите одну пустую страницу.  
- Текст «Tagged text at (100,200)» появится точно в заданных координатах.  
- Если открыть панель **Tags** (View → Show/Hide → Navigation Panes → Tags), вы найдёте узел `Span` под корнем, подтверждая, что PDF тегирован.
