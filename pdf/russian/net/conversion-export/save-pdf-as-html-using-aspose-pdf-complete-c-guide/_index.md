# Сохранить PDF как HTML с помощью Aspose.PDF – Полное руководство на C#

Когда‑нибудь вам нужно было **сохранить PDF как HTML**, но вы застряли на этапе «как‑это сделать»? Вы не одиноки. Во многих проектах — порталы документации, веб‑просмотры или шаблоны писем — преобразование PDF в чистый HTML экономит массу времени. Хорошая новость? С Aspose.PDF для .NET вы можете **конвертировать PDF в HTML** всего в несколько строк, и даже указать библиотеке встраивать CSS непосредственно в результат.

В этом руководстве мы пройдём всё, что вам нужно знать: точный код, почему каждый параметр важен, и несколько подводных камней, с которыми вы можете столкнуться. К концу вы сможете **экспортировать PDF в HTML** со встроенными стилями и без растровых изображений, готовый к вставке в любую веб‑страницу.

## Что вы узнаете

- Как создать простое консольное приложение C#, которое загружает PDF‑файл.  
- Какие флаги `HtmlSaveOptions` управляют растрированием изображений и встраиванием CSS.  
- Разница между **конвертацией PDF в HTML** и **экспортом PDF в HTML** на практике.  
- Советы по работе с большими документами, пользовательскими шрифтами и правами доступа к папкам.  
- Полный, готовый к запуску пример кода, который можно скопировать и сразу протестировать.

> **Требования:** У вас есть действующая лицензия Aspose.PDF для .NET (или вы согласны с водяным знаком оценки) и установлен .NET 6+. Другие сторонние пакеты не требуются.

## Сохранить PDF как HTML – пошаговый обзор

Ниже представлен общий процесс, который мы реализуем:

1. Загрузить исходный PDF‑файл.  
2. Создать экземпляр `HtmlSaveOptions` и настроить его для **встраивания CSS в HTML**.  
3. Вызвать `Document.Save` с этими параметрами, получив аккуратный файл `.html`.  

Вот и всё. Просто, не правда ли? Давайте разберём каждый шаг.

## Конвертация PDF в HTML: настройка проекта

Сначала создайте консольный проект (или вставьте код в любое существующее C#‑приложение). Единственный необходимый пакет NuGet — `Aspose.PDF`. Установите его через CLI:

```bash
dotnet add package Aspose.PDF
```

> **Почему Aspose.PDF?** Это полностью управляемая библиотека, поддерживающая широкий спектр возможностей PDF — шрифты, аннотации, векторную графику — без необходимости в Adobe Acrobat или внешних инструментах. Это делает **экспорт PDF в HTML** надёжным и быстрым.

После установки пакета добавьте обычные директивы `using`:

```csharp
using System;
using Aspose.Pdf;
```

## Экспорт PDF в HTML: настройка HtmlSaveOptions

Волшебство происходит в `HtmlSaveOptions`. Два флага особенно важны для чистого HTML‑вывода:

| Option          | Что делает                                                               | Рекомендуемое значение |
|-----------------|---------------------------------------------------------------------------|--------------------------|
| `RasterImages` | Когда **true**, все изображения конвертируются в растровые файлы (PNG/JPEG). | `false` – сохранять их как векторы |
| `EmbedCss`      | Встраивает сгенерированный CSS непосредственно в тег `<style>` в HTML‑файле. | `true` – избегает внешних CSS‑файлов |

Установка `RasterImages` в `false` предотвращает преобразование векторной графики в тяжёлые растровые файлы, что делает HTML лёгким и пригодным для поиска. Включение `EmbedCss` отвечает на вопрос «**как встраивать CSS в HTML**», делая результат самодостаточным — идеально для тел писем или одностраничных превью.

### Как встраивать CSS в HTML при конвертации PDF

Вот фрагмент кода, который настраивает эти параметры:

```csharp
var htmlSaveOptions = new HtmlSaveOptions
{
    RasterImages = false, // keep images as vectors
    EmbedCss = true       // embed CSS directly into the HTML
};
```

Вы можете задаться вопросом: *Что если мне нужен внешний CSS для большого сайта?* В этом случае просто установите `EmbedCss = false`, и библиотека создаст отдельный файл `.css` рядом с HTML. Для большинства сценариев «быстрого превью» встроенный подход наиболее удобен.

## Полный рабочий пример – сохранение PDF как HTML

Теперь соберём всё вместе. Скопируйте код ниже в `Program.cs` (или любой класс) и запустите его. Он прочитает `input.pdf` из той же папки, где находится исполняемый файл, и запишет `output.html` рядом.

```csharp
using System;
using Aspose.Pdf;

class Program
{
    static void Main()
    {
        // -------------------------------------------------
        // Step 1: Load the PDF document you want to convert
        // -------------------------------------------------
        // Replace "input.pdf" with the path to your source file.
        using (var pdfDocument = new Document("input.pdf"))
        {
            // -------------------------------------------------
            // Step 2: Configure HTML save options
            // -------------------------------------------------
            var htmlSaveOptions = new HtmlSaveOptions
            {
                // Prevent rasterization of vector graphics – keeps HTML light
                RasterImages = false,

                // Embed CSS directly into the generated HTML file
                EmbedCss = true
            };

            // -------------------------------------------------
            // Step 3: Export PDF to HTML using the configured options
            // -------------------------------------------------
            // The second argument tells Aspose where to write the HTML.
            pdfDocument.Save("output.html", htmlSaveOptions);
        }

        Console.WriteLine("✅ PDF successfully saved as HTML. Check output.html.");
    }
}
```

### Что ожидать

- `output.html` будет содержать разметку всей страницы, блок `<style>` со всем CSS и векторные изображения в формате SVG (если они были в PDF).  
- Отдельные файлы изображений или CSS не создаются, потому что мы установили `RasterImages = false` и `EmbedCss = true`.  
- Если PDF содержит шрифты, не установленные на машине, Aspose встроит их как Base64‑закодированные правила `@font-face`, обеспечивая сохранность визуального вида.

## Распространённые подводные камни при конвертации PDF в HTML

Даже простое API может подвести, если вы не знаете некоторых нюансов.

| Issue | Symptoms | Fix |
|-------|----------|-----|
| **Отсутствующая лицензия** | Вывод содержит водяной знак «Evaluation». | Примените вашу лицензию Aspose.PDF перед загрузкой документа (`License license = new License(); license.SetLicense("Aspose.PDF.lic");`). |
| **Большие PDF‑файлы** | Конвертация занимает >30 секунд или бросает `OutOfMemoryException`. | Используйте `pdfDocument.Compress();` перед сохранением или разбейте PDF на более мелкие части и конвертируйте их по отдельности. |
| **Пользовательские шрифты не отображаются** | Текст отображается общим запасным шрифтом. | Убедитесь, что шрифты установлены на хост‑машине, или встроите их, задав `htmlSaveOptions.FontEmbeddingMode = FontEmbeddingModes.EmbedAll;`. |
| **Разрешения файловой системы** | `UnauthorizedAccessException` при `Save`. | Запустите приложение с правом записи в целевую папку или выберите путь вроде `Path.Combine(Environment.GetFolderPath(Environment.SpecialFolder.Desktop), "output.html")`. |
| **Относительные пути к изображениям** (when `RasterImages = true`) | Изображения не отображаются в браузере. | Держите изображения и HTML в одной директории или используйте абсолютные URL, если размещаете изображения где‑то ещё. |

## Профессиональные советы для плавной конвертации

- **Пакетная конвертация:** Оберните блок `using` в цикл `foreach`, чтобы обработать всю папку PDF‑файлов.  
- **Тонкая настройка производительности:** Переиспользуйте один экземпляр `HtmlSaveOptions` для нескольких сохранений; объект недорогой в создании, но переиспользование избавляет от лишних выделений памяти.  
- **Тестирование результата:** Откройте сгенерированный HTML в Chrome DevTools и проверьте блок `<style>`. Если видите огромные строки Base64, это обычно означает, что шрифты были встроены — подходит для email, но может быть тяжеловесным для веб‑сценариев.  
- **Проверка версии:** Приведённый код работает с Aspose.PDF for .NET 23.7 и новее. Если у вас более старая версия, имена свойств могут немного отличаться (`HtmlSaveOptions.RasterImages` присутствует уже в многих релизах).
