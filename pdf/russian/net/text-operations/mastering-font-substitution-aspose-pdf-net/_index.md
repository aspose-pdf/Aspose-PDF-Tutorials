## Руководство по внедрению

### Реализация обработчиков замены шрифтов

**Обзор**
Замена шрифтов происходит, когда документ PDF ссылается на недоступный шрифт. Обрабатывая эти события, вы можете эффективно регистрировать и управлять этими изменениями.

#### Шаг 1: Настройте свою среду
Создайте новый проект C# в предпочитаемой вами IDE. Добавьте пакет Aspose.PDF, как описано выше.

#### Шаг 2: Создание обработчика событий замены шрифта

Добавьте обработчик событий для отслеживания замены шрифтов:

```csharp
using System;
using Aspose.Pdf;

namespace AsposePdfExamples
{
    public class GetWarningsForFontSubstitution
    {
        public static void Run()
        {
            // Путь к каталогу документов.
            string dataDir = RunExamples.GetDataDir_AsposePdf_WorkingDocuments();

            Document doc = new Document(dataDir + "input.pdf");

            // Подпишитесь на мероприятие FontSubstitution
            doc.FontSubstitution += OnFontSubstitution;
        }

        static void OnFontSubstitution(Font oldFont, Font newFont)
        {
            Console.WriteLine($"Font '{oldFont.FontName}' was substituted with '{newFont.FontName}'.");
        }
    }
}
```

**Объяснение:**
- The `OnFontSubstitution` Метод регистрирует события замены шрифтов. Это имеет решающее значение для отслеживания и отладки проблем, вызванных отсутствием шрифтов.
- Обработчик событий получает два параметра: `oldFont` и `newFont`, представляющие исходный и замененный шрифты соответственно.

### Советы по устранению неполадок
- Убедитесь, что путь к входному PDF-файлу правильный и доступный.
- Если события замены шрифтов не срабатывают, убедитесь, что ваш документ содержит шрифты, требующие замены.
