pdfPage <= pdfDocument1.getPages().size(); pdfPage++) {\n    // Create a new document for each individual page\n    Document newDocument = new Document();\n    \n    // Add the current page to the new document\n    newDocument.getPages().add(pdfDocument1.getPages().get_Item(pdfPage));\n    \n    // Save the new document as a separate PDF file named according to its page number\n    newDocument.save(\"YOUR_OUTPUT_DIRECTORY/page_\" + pdfPage + \".pdf\");\n}\n```\n\n**Wyjaśnienie:**\N- **`dataDir`:** Ścieżka do katalogu, w którym przechowywany jest plik PDF.\n- **`newDocument`:** Świeży przykład `Document` dla każdego podziału strony.\n- **`get_Item(pdfPage)`:** Pobiera określoną stronę z oryginalnego dokumentu.\n\n### Wskazówki dotyczące rozwiązywania problemów\n- Upewnij się, że ścieżki do katalogów istnieją i mają odpowiednie uprawnienia.\n- Sprawdź zgodność wersji Aspose.PDF, aby uniknąć problemów w czasie wykonywania.\n\n## Zastosowania praktyczne\n\nPodział plików PDF na pojedyncze strony może być nieoceniony w różnych scenariuszach:\n\n1. **Udostępnianie dokumentów**: Łatwe udostępnianie określonych sekcji bez konieczności wysyłania całych dokumentów.\n2. **Przetwarzanie danych**:Ułatw automatyczne przetwarzanie poprzez obsługę jednej strony na raz.\n3. **Zarządzanie plikami**Organizuj duże pliki PDF, aby ułatwić nawigację i archiwizację.\n\n## Zagadnienia dotyczące wydajności\n\nPodczas pracy z obszernymi plikami PDF optymalizacja wydajności jest kluczowa:\n\n- Używaj odpowiednich technik zarządzania pamięcią, aby wydajnie obsługiwać duże dokumenty.\n- Monitoruj wykorzystanie zasobów, aby zapobiegać powstawaniu wąskich gardeł w aplikacji.\n- Wykorzystuj wbudowane metody Aspose.PDF, aby zwiększyć szybkość przetwarzania.\n\n## Wnioski\n\nTeraz wiesz, jak podzielić dokument PDF na pojedyncze strony, używając Aspose.PDF dla Java.
