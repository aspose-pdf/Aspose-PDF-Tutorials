## 什麼是 PDF 可選內容群組？
PDF 可選內容群組（OCG）是 PDF 層背後的技術術語。它們允許您將頁面物件分組，使檢視器能獨立切換每個群組的可見性。這使得建立多語言 PDF、浮水印或互動表單成為可能，於不同條件下顯示不同內容。

## 為什麼使用 PDF 可選內容群組？
- **動態呈現：** 根據使用者偏好或裝置能力顯示或隱藏內容。  
- **減少檔案大小：** 一個 PDF 可包含多個內容版本，無需重複整份文件。  
- **提升可及性：** 將 OCG 與標籤結合，並執行 **PDF 可及性驗證**，改善螢幕閱讀器的導覽。  
- **改善 SEO：** 具標籤的分層 PDF 更易於搜尋引擎索引，特別是結合適當的中繼資料時。  

## 什麼是「使用 Java 建立 PDF 層」？
在 Java 中建立 PDF 層即是使用 Aspose.PDF API 定義可選內容群組（OCG），讓 PDF 檢視器能開啟或關閉。此技術非常適合多語言文件、浮水印或互動表單，在不同條件下顯示不同內容。

## 常見問題

**Q: 我可以在 PDF 建立後新增或移除層嗎？**  
A: 可以。Aspose.PDF 允許您隨時修改可選內容群組，實現動態更新而無需重新建立整個文件。

**Q: PDF 層會如何影響檔案大小？**  
A: 層共用相同的頁面資源，與為每個版本存放獨立 PDF 相比，通常只會增加少量開銷。

**Q: 所有檢視器都支援 PDF 層嗎？**  
A: 大多數現代 PDF 檢視器（Adobe Acrobat、Foxit、PDF.js）支援可選內容群組，但較舊或輕量的檢視器可能會忽略它們。

**Q: 使用層需要設定特殊權限嗎？**  
A: 不需要特殊權限；層是 PDF 規範的一部分，與其他頁面內容一樣處理。

**Q: 我可以將層與 PDF/UA 標籤結合以提升可及性嗎？**  
A: 當然可以。您可以為每個層的內容加上標籤，讓螢幕閱讀器更有效地導覽分層文件。

**Q: PDF 可及性驗證在分層 PDF 中如何運作？**  
A: Aspose.PDF 會對整個文件（包括每個可選內容群組）執行可及性驗證檢查，以確保符合 PDF/UA 標準。

**Q: 渲染大量層會影響效能嗎？**  
A: 渲染大量 OCG 可能會增加處理時間，但 Aspose.PDF 會優化資源共享，使大多數使用情境的效能仍在可接受範圍。
