# تعيين جافا سكريبت

## مقدمة

إنشاء ملفات PDF ديناميكية وتفاعلية يُحسّن تجربة المستخدم بشكل ملحوظ، خاصةً عند دمج النماذج والحقول داخل المستند. إحدى المكتبات القوية التي تُمكّن من ذلك هي Aspose.PDF لـ .NET. في هذه المقالة، سنتعمق في إعداد JavaScript لحقول النماذج باستخدام Aspose.PDF، لضمان جودة ملفات PDF، ليس فقط من حيث المظهر، بل وفعاليتها أيضًا.

## المتطلبات الأساسية

قبل أن ننتقل إلى البرمجة، دعنا نتأكد من أن لديك كل ما تحتاجه لمتابعة الأمر بسلاسة:

- Visual Studio (أو أي .NET IDE): تأكد من تثبيته وإعداده بشكل صحيح.
  
- مكتبة Aspose.PDF: ستحتاج إلى أحدث إصدار من هذه المكتبة. يمكنك تنزيله. [هنا](https://releases.aspose.com/pdf/net/).

- المعرفة الأساسية بلغة C#: ستساعدك المعرفة ببرمجة C# على فهم مقتطفات التعليمات البرمجية بشكل أفضل.

- ملفات PDF: يجب أن يكون لديك ملف PDF جاهز للاختبار. في مثالنا، سنستخدم ملفًا باسم `SetJavaScript.pdf`.

- دليل مستنداتك: اعرف مكان تخزين ملفات مستنداتك. سنشير إلى هذا المسار في الكود الخاص بنا.

بعد أن تُجهّز هذه المتطلبات الأساسية، ما هي الأدوات التي سنستخدمها؟ لنستكشف إمكانيات Aspose.PDF.

## استيراد الحزم

للبدء، عليك تضمين مساحات الأسماء اللازمة في مشروع C#. افتح ملف C# الرئيسي وأضف عبارات الاستيراد التالية:

```csharp
using System;
using System.IO;
using Aspose.Pdf.Forms;
using Aspose.Pdf.Annotations;
using Aspose.Pdf;
```

توفر هذه المساحات الاسمية إمكانية الوصول إلى ملفات PDF والوظائف المرتبطة بالنماذج داخل مكتبة Aspose.PDF.

هل أنت مستعد لجعل ملف PDF الخاص بك تفاعليًا؟ هيا بنا نشرح الأمر خطوة بخطوة!

## الخطوة 1: تحديد مسار المستند

أولاً، علينا تحديد موقع ملف PDF. هذا يُمهّد الطريق لكل ما يلي. إليك كيفية القيام بذلك:

```csharp
string dataDir = "YOUR DOCUMENT DIRECTORY";
```

يستبدل `"YOUR DOCUMENT DIRECTORY"` مع المسار الفعلي لملف PDF الخاص بك. تخيل هذا كتحديد إحداثيات خريطة الكنز - عليك معرفة مكان علامة "X"!

## الخطوة 2: تحميل مستند PDF

بمجرد أن نقوم بتحديد الدليل، سنقوم بتحميل ملف PDF الخاص بنا. 

```csharp
Document doc = new Document(dataDir + "SetJavaScript.pdf");
```

يفتح هذا السطر ملف PDF المحدد ويجهزه للتعامل معه. 

## الخطوة 3: الوصول إلى حقل النموذج

بعد ذلك، نريد الوصول إلى حقل النموذج حيث سنطبق JavaScript الخاص بنا. 

```csharp
TextBoxField field = (TextBoxField)doc.Form["textbox1"];
```

هنا، نفترض أن هناك مربع نص في ملف PDF الخاص بك يسمى `textbox1`إذا لم يكن لديك حقل بهذا الاسم، فيمكنك إعادة تسميته أو تعديل الكود وفقًا لذلك. 

## الخطوة 4: إعداد إجراءات JavaScript

الآن، لنُضِف بعض الوظائف إلى مربع النص! سنُعِدّ إجراءات جافا سكريبت التي سيتم تفعيلها عند أحداث مُحددة. 

```csharp
field.Actions.OnModifyCharacter = new JavascriptAction("AFNumber_Keystroke(2, 1, 1, 0, \"\", true)");
field.Actions.OnFormat = new JavascriptAction("AFNumber_Format(2, 1, 1, 0, \"\", true)");
```

وإليك ما يحدث:
- OnModifyCharacter: تُحدد هذه الدالة في جافا سكريبت كيفية عمل الحقل عند تعديل حرف. في هذه الحالة، تسمح بوضع نقطتين عشريتين بعد الرقم بدون فاصل.
- OnFormat: هذا يضمن أنه عندما يقوم المستخدم بتنسيق الرقم، فإنه يلتزم بنفس القاعدة.

من خلال إعداد هذه الإجراءات، فإننا نعطي مربع النص الخاص بنا شخصية بشكل أساسي - مثل تعليمه حركة رقص!

## الخطوة 5: تهيئة قيمة الحقل

بعد ذلك، دعونا نعطي مربع النص الخاص بنا نقطة بداية عن طريق تعيين قيمة أولية. 

```csharp
field.Value = "123";
```

يُعيّن هذا السطر "١٢٣" كقيمة مُعبأة مُسبقًا في مربع النص. يشبه الأمر تجهيز خشبة المسرح لعرضٍ ما.

## الخطوة 6: حفظ ملف PDF المعدل

وأخيرًا، نحتاج إلى حفظ مستندنا بعد إجراء كل هذه التغييرات.

```csharp
dataDir = dataDir + "Restricted_out.pdf";
doc.Save(dataDir);
```

يؤدي هذا إلى تحديث الملف الأصلي بالتغييرات التي أجريتها وحفظه باسم `Restricted_out.pdf`فكر في هذا الأمر باعتباره تحديد مصير ملف PDF الخاص بنا - بمجرد حفظه، يصبح جاهزًا للعالم!

## الخطوة 7: تأكيد النجاح

وأخيرا، دعونا نتحقق من أن كل شيء يسير بسلاسة. 

```csharp
Console.WriteLine("\nJavaScript on form field setup successfully.\nFile saved at " + dataDir);
```

إن تشغيل هذه الرسالة سوف يطمئنك إلى أن العملية قد اكتملت بنجاح، تمامًا مثل تلقي تصفيق من الجمهور بعد أداء رائع.
