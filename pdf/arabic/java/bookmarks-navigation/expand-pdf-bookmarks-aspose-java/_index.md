إذا فتحت ملف PDF ضخم واضطررت إلى النقر عبر لوحة المخطط يدويًا، فأنت تعرف مدى الإحباط الذي قد يسببه ذلك. في هذا **Aspose PDF Java tutorial** سنظهر لك **كيفية توسيع العلامات المرجعية** تلقائيًا بحيث يرى القراء جدول المحتويات الكامل في اللحظة التي يفتح فيها الملف. ستتعلم كيفية تحميل PDF، ضبط وضع صفحة العارض، توسيع كل مدخل في المخطط، وحفظ النتيجة — كل ذلك ببضع أسطر من كود Java فقط.

توسيع العلامات المرجعية في PDF تلقائيًا يحسن كفاءة التنقل للمستخدمين النهائيين، مما يسمح لهم برؤية الهيكل الكامل للمستند دون نقرات إضافية. هذه الميزة ذات قيمة خاصة في المستندات الطويلة حيث يكون العثور على الأقسام بسرعة أمرًا حيويًا. من خلال عرض لوحة المخطط عند الفتح، يمكن للقراء القفز مباشرة إلى الفصل أو البند المطلوب، مما يقلل الوقت المستغرق في التمرير والبحث.

يفتح توسيع العلامات المرجعية تلقائيًا وقت المستخدم، خاصة في:

## إعداد Aspose.PDF للـ Java
أضف مكتبة Aspose.PDF إلى ملف البناء الخاص بك.

**Maven**  
```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-pdf</artifactId>
    <version>25.3</version>
</dependency>
```

**Gradle**  
```gradle
implementation 'com.aspose:aspose-pdf:25.3'
```

## كيف يتم تحميل مستند PDF؟
حمّل ملف PDF الخاص بك إلى كائن Aspose `Document`. هذه السطر الواحد ينشئ تمثيلًا في الذاكرة للملف المصدر، مما يتيح لك قراءة وتعديل هيكله. تمثل فئة `Document` ملف PDF في الذاكرة، وتوفر الوصول إلى صفحاته، مخططاته، وعناصر أخرى، لتكون الكائن المركزي لجميع عمليات PDF.

```java
// Load an existing PDF document from your specified directory
document doc = new Document("YOUR_DOCUMENT_DIRECTORY/input.pdf");
```

## كيف يتم ضبط وضع الصفحة لتوسيع العلامات المرجعية؟
اضبط وضع صفحة العارض إلى `UseOutlines` بحيث يعرض قارئ PDF لوحة العلامات المرجعية تلقائيًا عند فتح الملف. تُكوّن طريقة `setPageMode` سلوك العارض الافتراضي، ويشير الثابت `PageMode.UseOutlines` إلى عارض PDF لعرض لوحة المخطط عند الإطلاق.

```java
doc.setPageMode(PageMode.UseOutlines);
```

## كيف يتم توسيع جميع العلامات المرجعية في PDF؟
تجول عبر كل مدخل في المخطط واضبط خاصية `Open` إلى `true`. هذا يجعل كل علامة مرجعية تظهر مُوسعة افتراضيًا. تحتفظ فئة `OutlineItemCollection` بجميع مدخلات العلامات المرجعية؛ من خلال تعيين كل عنصر كـ مفتوح، سيعرض عارض PDF التسلسل الهرمي الكامل دون الحاجة لتفاعل المستخدم.

```java
// Iterate over each bookmark and set its state to open
for (int counter = 1; counter <= doc.getOutlines().size(); counter++) {
    doc.getOutlines().get_Item(counter).setOpen(true);
}
```

## كيف يتم حفظ PDF بعد توسيع العلامات المرجعية؟
اكتب المستند المعدل إلى ملف جديد (أو استبدل الأصلي). تُحفظ جميع التغييرات التي أجريتها على بنية PDF عبر طريقة `save()`. بعد استدعاء `save`، سيحتفظ PDF بحالة العلامات المرجعية الموسعة لأي عارض يفتحه.

```java
doc.save("YOUR_OUTPUT_DIRECTORY/Bookmarks_Expanded.pdf");
```

## اعتبارات الأداء
عند معالجة PDFs الكبيرة التي تحتوي على آلاف المخططات:

- حرّر كائن `Document` بعد الحفظ لتفريغ الذاكرة.  
- عالج المخططات على دفعات أصغر إذا كان المستند يحتوي على عدد كبير من المدخلات.  
- ضبط حجم كومة Java (`-Xmx`) وفقًا لحجم عملك؛ بالنسبة لمستند من 500 صفحة، غالبًا ما يكون `-Xmx2g` كافيًا.
