# C#’ta PDF’yi HTML’e Dönüştürme – Tam Programlama Kılavuzu

Web’de PDF içeriğini ağır görüntü varlıkları eklemeden göstermek istediğinizde **PDF’yi HTML’e dönüştürmek** için hangi API çağrılarının işe yaradığını hiç merak ettiniz mi? Tek başınıza değilsiniz—geliştiriciler bu sorunu sık sık yaşar.  

İyi haber? Birkaç satır kodla **C#’ta PDF belgesini yükleyebilir**, dönüşümü görüntüleri atacak şekilde yapılandırabilir ve ön uç için hafif bir HTML dosyası elde edebilirsiniz. Proje kurulumundan son HTML çıktısına kadar tüm süreci adım adım inceleyelim.

> **Neler elde edeceksiniz:** `sample.pdf` dosyasını okuyan, tüm görüntüleri kaldıran ve `sample-no-images.html` dosyasını yazan hazır‑çalışır bir konsol uygulaması. Ekstra dosya, sihir yok, sadece sade C#.

```bash
dotnet add package Aspose.PDF
```

## Adım 1: PDF Belgesini C#’ta Yükleyin

İlk yapmanız gereken PDF’i belleğe almak. Aspose.PDF’de bu, dosya yolunu vererek bir `Document` nesnesi oluşturmak kadar basittir.

```csharp
using Aspose.Pdf;

string pdfPath = Path.Combine(Environment.CurrentDirectory, "sample.pdf");

// Load the PDF document – this is the “load pdf document in c#” part.
Document pdfDoc = new Document(pdfPath);
```

> **Neden önemli:** PDF’i yüklemek, sayfalara, kaynaklara ve meta‑verilere erişmenizi sağlar. Bu adım olmadan hiçbir şeyi manipüle edemez, dönüştüremezsiniz.

---

## Adım 2: Görüntüleri Atlamak İçin HTML Kaydetme Seçeneklerini Yapılandırın

PDF’i HTML’e dönüştürdüğünüzde görüntüler genellikle ortaya çıkan dosyanın boyutunu şişirir. Sadece metin düzenine ihtiyacınız varsa `SkipImages` değerini `true` yapın. Diğer ayarları da değiştirebilirsiniz, ancak minimal bir örnek için sadece bu bayrağa odaklanacağız.

```csharp
using Aspose.Pdf.Saving;

// Set up HTML save options – we’ll skip images for a leaner output.
HtmlSaveOptions htmlOptions = new HtmlSaveOptions
{
    SkipImages = true,
    // Optional: you can control CSS, fonts, and other assets here.
    EmbedCss = true,
    EmbedFonts = false
};
```

> **İpucu:** `EmbedCss = true` stil tanımlarını aynı HTML dosyasında tutar, hızlı ön izlemeler için kullanışlıdır. Dış CSS tercih ederseniz `false` yapıp daha sonra bir stil sayfası ekleyebilirsiniz.

---

## Adım 3: PDF’i Görüntüsüz HTML Olarak Kaydedin

Belge yüklendi ve seçenekler ayarlandı, dönüşüm tek bir metod çağrısıyla gerçekleşir.

```csharp
string htmlPath = Path.Combine(Environment.CurrentDirectory, "sample-no-images.html");

// Perform the conversion – this is the core “convert pdf to html” operation.
pdfDoc.Save(htmlPath, htmlOptions);

Console.WriteLine($"Conversion complete! HTML saved to: {htmlPath}");
```

Programı çalıştırdığınızda, çıktı konumunu onaylayan bir konsol mesajı göreceksiniz. `sample-no-images.html` dosyasını herhangi bir tarayıcıda açın—orijinal PDF düzeni korunur, ancak tüm `<img>` etiketleri yoktur.

### Beklenen Çıktı

```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>sample</title>
    <style type="text/css">
        /* Inline CSS generated by Aspose.PDF */
    </style>
</head>
<body>
    <div class="page">
        <p>Welcome to the PDF‑to‑HTML demo.</p>
        <!-- No <img> elements appear here because SkipImages was true -->
    </div>
</body>
</html>
```

`<img>` etiketlerinin eksikliğine dikkat edin—tam olarak istediğimiz şey bu.

## Adım 4: Kenar Durumları ve Yaygın Sorular

### PDF vektörel grafikler içeriyorsa ne olur?

Vektörel grafikler SVG veya Canvas öğeleri olarak işlenir, **raster görüntü** olarak değil, bu yüzden `SkipImages` onları etkilemez. Bunları da kaldırmak isterseniz HTML’i sonradan işleyebilir veya `HtmlSaveOptions`’ın `RasterImages` bayrağını kullanabilirsiniz.

### Tek bir çalıştırmada birden fazla PDF’i dönüştürebilir miyim?

Tabii ki. Yükleme ve kaydetme mantığını bir klasördeki dosyalar üzerinde `foreach` döngüsüyle sarabilirsiniz:

```csharp
foreach (var file in Directory.GetFiles(@"C:\pdfs", "*.pdf"))
{
    var doc = new Document(file);
    var outPath = Path.ChangeExtension(file, ".html");
    doc.Save(outPath, htmlOptions);
}
```

### .NET Framework 4.7’de çalışır mı?

Evet, Aspose.PDF .NET Framework 4.0+’ı destekler. Proje hedefini değiştirmeniz yeterli, kod aynı kalır.

### Şifre korumalı PDF’ler nasıl ele alınır?

Şifreyi `Document` yapıcısına parametre olarak geçirin:

```csharp
Document protectedDoc = new Document(pdfPath, new LoadOptions { Password = "mySecret" });
```

Ardından aynı `Save` çağrısını yapın.

## Adım 5: Tam, Hazır‑Çalışır Örnek

Aşağıda yeni bir `.csproj` içine kopyalayıp yapıştırabileceğiniz eksiksiz bir konsol programı bulacaksınız. Tüm `using` ifadeleri, hata yönetimi ve yorumlar dahildir.

```csharp
// Program.cs
using System;
using System.IO;
using Aspose.Pdf;
using Aspose.Pdf.Saving;

class Program
{
    static void Main()
    {
        try
        {
            // 1️⃣ Load the PDF document – “load pdf document in c#”
            string pdfPath = Path.Combine(Environment.CurrentDirectory, "sample.pdf");
            Document pdfDoc = new Document(pdfPath);

            // 2️⃣ Configure conversion – skip images for a lean HTML file
            HtmlSaveOptions htmlOptions = new HtmlSaveOptions
            {
                SkipImages = true,
                EmbedCss = true,
                EmbedFonts = false
            };

            // 3️⃣ Save as HTML – the actual “convert pdf to html” step
            string htmlPath = Path.Combine(Environment.CurrentDirectory, "sample-no-images.html");
            pdfDoc.Save(htmlPath, htmlOptions);

            Console.WriteLine($"✅ Conversion successful! HTML file created at: {htmlPath}");
        }
        catch (Exception ex)
        {
            Console.Error.WriteLine($"❌ An error occurred: {ex.Message}");
        }
    }
}
```

Derleyip çalıştırın:

```bash
dotnet run
```

Başarı mesajını göreceksiniz ve HTML dosyası çalıştırılabilir dosyanızın yanına yerleştirilecektir.
