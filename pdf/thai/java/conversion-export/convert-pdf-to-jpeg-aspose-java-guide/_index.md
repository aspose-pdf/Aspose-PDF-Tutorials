## ฟีเจอร์ 1: แปลงทุกหน้าของ PDF เป็นภาพ JPEG

### “แปลงทุกหน้าของ PDF” หมายความว่าอะไร?
ฟีเจอร์นี้ทำการวนลูปผ่านทุกหน้าของเอกสาร PDF และบันทึกแต่ละหน้าเป็นไฟล์ JPEG แยกกัน เหมาะสำหรับสถานการณ์ **batch convert pdf jpeg** เช่น การสร้างอาร์ไคฟ์รูปภาพหรือการเตรียม thumbnail สำหรับแกลเลอรีบนเว็บ

### ขั้นตอนการทำงานทีละขั้นตอน

**ขั้นตอนที่ 1: โหลดเอกสาร PDF**  
เราจะเริ่มโดยสร้างอ็อบเจ็กต์ `Document` ที่เป็นตัวแทนของ PDF ต้นฉบับ

```java
Document pdfDocument = new Document(dataDir + "/input.pdf");
```

**ขั้นตอนที่ 2: วนลูปผ่านแต่ละหน้า**  
ลูป `for` ง่าย ๆ จะเดินทางผ่านดัชนีของทุกหน้า

```java
for (int pageCount = 1; pageCount <= pdfDocument.getPages().size(); pageCount++) {
    // Processing code here
}
```

**ขั้นตอนที่ 3: ตั้งค่าพารามิเตอร์การส่งออกและการแปลง**  
ตั้งค่า `JpegDevice` ด้วยความละเอียด 300 DPI และคุณภาพสูงสุด (100).

```java
Resolution resolution = new Resolution(300);
JpegDevice jpegDevice = new JpegDevice(resolution, 100);
```

**ขั้นตอนที่ 4: แปลงแต่ละหน้าเป็น JPEG**  
ภายในลูป เราจะเขียนแต่ละหน้าเป็นไฟล์ JPEG โดยใช้บล็อก try‑with‑resources เพื่อให้แน่ใจว่าการสตรีมถูกปิดอย่างถูกต้อง.

```java
try (OutputStream imageStream = new FileOutputStream(outputDir + "/Converted_Image" + pageCount + ".jpg")) {
    jpegDevice.process(pdfDocument.getPages().get_Item(pageCount), imageStream);
}
```

#### ข้อผิดพลาดทั่วไปและเคล็ดลับ
- **ความถูกต้องของพาธ:** ตรวจสอบค่า `dataDir` และ `outputDir` อีกครั้ง; พาธสัมพัทธ์มักทำให้วางผิดได้ง่าย  
- **สิทธิ์ไฟล์:** ตรวจสอบให้แน่ใจว่าโปรเซส Java มีสิทธิ์เขียนไปยังโฟลเดอร์ผลลัพธ์  
- **การใช้หน่วยความจำ:** สำหรับ PDF ขนาดใหญ่มาก ให้พิจารณาประมวลผลหน้าเป็นชุดเล็ก ๆ หรือเพิ่มขนาด heap ของ JVM

## ฟีเจอร์ 2: แปลงหน้า PDF หนึ่งหน้าเป็นภาพ JPEG

### เมื่อใดที่คุณต้องการ “แปลงหน้า PDF เดียว”?
บางครั้งคุณอาจต้องการภาพสแนปช็อตของหน้าที่เฉพาะ — เช่น หน้าปก แผนภูมิ หรือหน้าลายเซ็น ฟีเจอร์นี้แสดงวิธีการเลือกหน้าเดียว

### ขั้นตอนการทำงานทีละขั้นตอน

**ขั้นตอนที่ 1: โหลดเอกสาร PDF**  
เราจะเริ่มโดยสร้างอ็อบเจ็กต์ `Document` ที่เป็นตัวแทนของ PDF ต้นฉบับ

```java
Document pdfDocument = new Document(dataDir + "/input.pdf");
```

**ขั้นตอนที่ 2: ตั้งค่าการสตรีมเอาต์พุตและพารามิเตอร์การแปลง**  
ใช้การตั้งค่า `Resolution` และ `JpegDevice` เดิมซ้ำ

```java
Resolution resolution = new Resolution(300);
JpegDevice jpegDevice = new JpegDevice(resolution, 100);
```

**ขั้นตอนที่ 3: แปลงหน้าที่ระบุเป็น JPEG**  
ที่นี่เราจะแปลงเฉพาะหน้าแรก (เปลี่ยนดัชนีเพื่อเลือกหน้าที่ต้องการอื่น).

```java
try (OutputStream imageStream = new FileOutputStream(outputDir + "/Converted_Image.jpg")) {
    jpegDevice.process(pdfDocument.getPages().get_Item(1), imageStream);
}
```

#### รายการตรวจสอบอย่างรวดเร็ว
- ตรวจสอบว่ามีหมายเลขหน้าที่ต้องการ (`pdfDocument.getPages().size()`).  
- จัดการ `IndexOutOfBoundsException` หากดัชนีหน้านอกช่วง.  
- ใช้รูปแบบ try‑with‑resources เดียวกันเพื่อหลีกเลี่ยงการรั่วของทรัพยากร.

## การประยุกต์ใช้งานจริง

การแปลงหน้าของ PDF เป็นภาพ JPEG สามารถเป็นประโยชน์ในหลายสถานการณ์:

2. **การเก็บถาวร:** เก็บภาพสแนปช็อตของเอกสารในกรณีที่ไม่มีโปรแกรมอ่าน PDF.  
3. **การแชร์เนื้อหา:** ฝังหน้าของ PDF อย่างรวดเร็วในอีเมล โพสต์โซเชียลมีเดีย หรือสไลด์เด็ค.

การผสานฟังก์ชันนี้เข้าไปใน CMS, ห้องสมุดดิจิทัล หรือกระบวนการรายงานอัตโนมัติสามารถปรับปรุงประสบการณ์ผู้ใช้ได้อย่างมาก

## การพิจารณาประสิทธิภาพ

- **การตั้งค่าความละเอียด:** DPI สูงให้ภาพคมชัดขึ้นแต่เพิ่มเวลาในการประมวลผลและขนาดไฟล์ ปรับตามกรณีการใช้งานของคุณ.  
- **การจัดการหน่วยความจำ:** ใช้ try‑with‑resources (ตามที่แสดง) เพื่อปล่อยสตรีมอย่างรวดเร็วและตรวจสอบ heap ของ JVM เมื่อจัดการ PDF ขนาดใหญ่.  
- **การประมวลผลแบบขนาน:** สำหรับงานแบชขนาดใหญ่ พิจารณาประมวลผลหน้าด้วย parallel streams แต่ต้องระวังความปลอดภัยของเธรดกับ `JpegDevice`.

**Q: การแปลงจัดการกับกราฟิกเวกเตอร์อย่างไร?**  
A: องค์ประกอบเวกเตอร์จะถูกแรสเตอร์ที่ DPI ที่กำหนด เพื่อรักษาความเที่ยงตรงของภาพใน JPEG ที่ได้

**Q: ถ้าฉันต้องการพื้นหลังโปร่งใสจะทำอย่างไร?**  
A: JPEG ไม่รองรับความโปร่งใส; เปลี่ยนเป็น PNG หรือ TIFF หากต้องการช่องอัลฟ่า

**Q: เวอร์ชันทดลองเพิ่มลายน้ำหรือไม่?**  
A: เวอร์ชันทดลองฟรีจะเพิ่มลายน้ำขนาดเล็กบนหน้าแรกของแต่ละภาพที่สร้างขึ้น เวอร์ชันที่มีใบอนุญาตจะลบลายน้ำทั้งหมด
