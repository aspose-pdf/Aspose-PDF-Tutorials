pdfPage <= pdfDocument1.getPages().size(); pdfPage++) {\n    // Create a new document for each individual page\n    Document newDocument = new Document();\n    \n    // Add the current page to the new document\n    newDocument.getPages().add(pdfDocument1.getPages().get_Item(pdfPage));\n    \n    // Save the new document as a separate PDF file named according to its page number\n    newDocument.save(\"YOUR_OUTPUT_DIRECTORY/page_\" + pdfPage + \".pdf\");\n}\n```Bahasa Indonesia: tidak**Penjelasan:**\N- **`dataDir`:** Jalur direktori tempat PDF masukan Anda disimpan.\n- **`newDocument`:** Contoh baru dari `Document` untuk setiap halaman yang dibagi.\n- **`get_Item(pdfPage)`:** Mengambil halaman tertentu dari dokumen asli.\n\n### Tips Pemecahan Masalah\n- Pastikan jalur direktori tersedia dan memiliki izin yang sesuai.\n- Verifikasi kompatibilitas versi Aspose.PDF untuk menghindari masalah runtime.\n\n## Aplikasi Praktis\n\nMembagi PDF ke dalam halaman-halaman tersendiri dapat sangat berguna dalam berbagai skenario:\n\n1. **Berbagi Dokumen**: Bagikan bagian tertentu dengan mudah tanpa mengirim seluruh dokumen.\n2. **Pengolahan data**: Memfasilitasi pemrosesan otomatis dengan menangani satu halaman dalam satu waktu.\n3. **Manajemen Berkas**Atur file PDF yang besar untuk memudahkan navigasi dan pengarsipan.\n\n## Pertimbangan Kinerja\n\nSaat menangani PDF yang besar, mengoptimalkan kinerja adalah kuncinya:\n\n- Gunakan teknik manajemen memori yang tepat untuk menangani dokumen besar secara efisien.\n- Pantau penggunaan sumber daya untuk mencegah kemacetan dalam aplikasi Anda.\n- Manfaatkan metode bawaan Aspose.PDF untuk meningkatkan kecepatan pemrosesan.\n\n## Kesimpulan\n\nAnda sekarang telah mempelajari cara membagi dokumen PDF menjadi beberapa halaman individual menggunakan Aspose.PDF untuk Java.
