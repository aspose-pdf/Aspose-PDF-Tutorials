## Οδηγός Εφαρμογής

### Κατανόηση της δημιουργίας διάταξης N-up

Η διάταξη N-up σάς επιτρέπει να ενοποιήσετε πολλές σελίδες σε μία μόνο σελίδα με πολλές στήλες και γραμμές. Αυτό είναι ιδιαίτερα χρήσιμο για τη μείωση του μεγέθους του αρχείου και του κόστους εκτύπωσης.

#### Βήμα 1: Αρχικοποίηση του PdfFileEditor

Ξεκινήστε δημιουργώντας μια παρουσία του `PdfFileEditor`:

```csharp
using Aspose.Pdf.Facades;

class MakeNUpUsingPageSizeAndPaths
{
    public static void Run()
    {
        // Ορίστε τις διαδρομές του εγγράφου προέλευσης και εξόδου
        string dataDir = @"YOUR_DOCUMENT_DIRECTORY";
        string outputPath = @"YOUR_OUTPUT_DIRECTORY";

        using (PdfFileEditor pdfEditor = new PdfFileEditor())
```

#### Βήμα 2: Ρύθμιση παραμέτρων N-up

Καθορίστε τον αριθμό των γραμμών και των στηλών για τη διάταξή σας, καθώς και το επιθυμητό μέγεθος σελίδας:

```csharp
int nUpRows = 2;   // Αριθμός γραμμών στη νέα διάταξη
int nUpColumns = 3; // Αριθμός στηλών στη νέα διάταξη
PageSize pageSize = PageSize.A5; // Μέγεθος σελίδας για κάθε σελίδα εξόδου
```

#### Βήμα 3: Εκτέλεση λειτουργίας MakeNUp

Χρησιμοποιήστε το `MakeNUp` μέθοδος για την αναδιάταξη των σελίδων:

```csharp
pdfEditor.MakeNUp(
    dataDir + "MultiplePages.pdf",         // Διαδρομή αρχείου εισόδου
    outputPath + "MakeNUpUsingPageSizeAndPaths_out.pdf", // Διαδρομή αρχείου εξόδου
    nUpRows,                              // Αριθμός γραμμών στη νέα διάταξη
    nUpColumns,                           // Αριθμός στηλών στη νέα διάταξη
    pageSize                             // Μέγεθος σελίδας για κάθε σελίδα στο έγγραφο εξόδου
);
}
```

#### Εξήγηση

- **Παράμετροι**: Το `MakeNUp` Η μέθοδος απαιτεί διαδρομές αρχείων εισόδου και εξόδου, αριθμό γραμμών και στηλών και μέγεθος σελίδας.
- **Επιστρεφόμενες τιμές**Επιστρέφει μια λογική τιμή που υποδεικνύει την επιτυχία ή την αποτυχία της λειτουργίας.
