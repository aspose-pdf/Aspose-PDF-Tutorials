# Δημιουργία Υπογεγραμμένου PDF σε C# – Πλήρης Οδηγός Προγραμματισμού

Έχετε ποτέ χρειαστεί να **δημιουργήσετε υπογεγραμμένα PDF** αρχεία από μια εφαρμογή .NET αλλά δεν ήξερατε από πού να ξεκινήσετε; Δεν είστε μόνοι. Σε πολλές επιχειρησιακές ροές εργασίας, ένα υπογεγραμμένο PDF είναι το τελικό κομμάτι που κλείνει μια σύμβαση, επικυρώνει ένα τιμολόγιο ή συμμορφώνεται με κανονισμούς. Τα καλά νέα; Με μερικές γραμμές C# και Aspose.Pdf μπορείτε να **προσθέσετε ψηφιακή υπογραφή** σε οποιοδήποτε PDF σε ελάχιστο χρόνο.

Σε αυτό το tutorial θα περάσουμε βήμα προς βήμα τις ακριβείς ενέργειες **πώς να υπογράψετε PDF** χρησιμοποιώντας ένα πιστοποιητικό PFX, γιατί μια αποσπασμένη υπογραφή PKCS#7 είναι συχνά η ασφαλέστερη επιλογή, και πώς να **υπογράψετε PDF με πιστοποιητικό** χωρίς να καταστρέψετε το αρχικό έγγραφο. Στο τέλος θα έχετε ένα έτοιμο παράδειγμα που δημιουργεί ένα υπογεγραμμένο PDF, μαζί με συμβουλές για κοινές περιπτώσεις.

## Τι Θα Χρειαστείτε

- **Aspose.Pdf for .NET** (v23.9 ή νεότερο). Το πακέτο NuGet ονομάζεται `Aspose.Pdf`.
- Ένα **πιστοποιητικό PKCS#12 (.pfx)** που περιέχει ιδιωτικό κλειδί που επιτρέπεται να χρησιμοποιήσετε για υπογραφή.
- .NET 6+ runtime (ο κώδικας λειτουργεί επίσης σε .NET Framework 4.7+).
- Ένα απλό PDF (`toSign.pdf`) που θέλετε να προστατέψετε.

Καμία επιπλέον βιβλιοθήκη, καμία εξωτερική υπηρεσία — μόνο τα παραπάνω στοιχεία.

## Βήμα 1 – Φόρτωση του PDF που Θέλετε να Υπογράψετε

Πριν μπορέσετε να εφαρμόσετε οποιαδήποτε υπογραφή, χρειάζεστε ένα αντικείμενο `Document` που αντιπροσωπεύει το αρχείο προέλευσης.

```csharp
using Aspose.Pdf;

// Load the PDF that will be signed
using var pdfDocument = new Document(@"C:\PDFs\toSign.pdf");
```

*Γιατί είναι σημαντικό:* `Document` είναι το σημείο εισόδου για όλες τις λειτουργίες PDF στο Aspose. Χρησιμοποιώντας μια δήλωση `using` διασφαλίζουμε ότι ο χειριστής του αρχείου απελευθερώνεται άμεσα, αποφεύγοντας σφάλματα “αρχείο σε χρήση” όταν προσπαθούμε να αποθηκεύσουμε την υπογεγραμμένη έκδοση.

## Βήμα 2 – Ρύθμιση του Διαχειριστή Υπογραφής

Το Aspose παρέχει μια ειδική διεπαφή που ονομάζεται `PdfFileSignature` η οποία ξέρει πώς να ενσωματώνει υπογραφές χωρίς να καταστρέφει το υπόλοιπο αρχείο.

```csharp
using Aspose.Pdf.Facades;

// Create a signature handler for the loaded document
using var pdfSigner = new PdfFileSignature(pdfDocument);
```

*Συμβουλή:* Αν σκοπεύετε να προσθέσετε πολλαπλές υπογραφές αργότερα, διατηρήστε το `isAppendMode` σε `true` (θα το κάνουμε στο επόμενο βήμα). Αυτό ενημερώνει τη βιβλιοθήκη να προσθέσει μια νέα αυξητική ενημέρωση αντί να ξαναγράψει ολόκληρο το αρχείο.

## Βήμα 3 – Προετοιμασία Αποσπασμένης Υπογραφής PKCS#7

Μια **αποσπασμένη υπογραφή PKCS#7** αποθηκεύει το hash του εγγράφου ξεχωριστά από τα δεδομένα του πιστοποιητικού, καθιστώντας την επαλήθευση πιο εύκολη και διατηρώντας το αρχικό PDF άθικτο. Ακολουθεί η διαμόρφωση με SHA‑512, το οποίο είναι ισχυρότερο από το προεπιλεγμένο SHA‑256.

```csharp
using Aspose.Pdf.Forms;

// Prepare a PKCS#7 detached signature using SHA‑512 as the digest algorithm
var pkcsSignature = new PKCS7Detached(
    @"C:\Certificates\mycert.pfx",   // certificate file
    "pwd",                           // certificate password
    DigestHashAlgorithm.Sha512);     // explicit digest algorithm
```

*Γιατί SHA‑512;* Πολλά πρότυπα συμμόρφωσης (π.χ., EU eIDAS) συνιστούν τουλάχιστον 256‑bit hashes, και το SHA‑512 προσφέρει άνετο περιθώριο χωρίς αισθητή επίπτωση στην απόδοση.

## Βήμα 4 – Εφαρμογή της Ψηφιακής Υπογραφής σε Συγκεκριμένη Σελίδα

Τώρα προσθέτουμε πραγματικά **ψηφιακή υπογραφή** στο PDF. Μπορείτε να επιλέξετε οποιαδήποτε σελίδα και οποιοδήποτε ορθογώνιο· το ορθογώνιο καθορίζει πού θα τοποθετηθεί η ορατή εμφάνιση της υπογραφής.

```csharp
using System.Drawing;

// Apply the digital signature to page 1 at the desired rectangle
pdfSigner.Sign(
    pageNumber: 1,                     // page index starts at 1
    isAppendMode: true,                // keep existing signatures intact
    signatureRectangle: new Rectangle(100, 100, 200, 150),
    pkcsSignature);
```

*Συχνή ερώτηση:* “Τι γίνεται αν δεν θέλω ορατή υπογραφή?”  
Απλώς περάστε `null` για το `signatureRectangle` και η βιβλιοθήκη θα δημιουργήσει μια αόρατη (χωρίς σημείωση) υπογραφή, χρήσιμη για διαδικασίες backend.

## Βήμα 5 – Αποθήκευση του Υπογεγραμμένου PDF

Τέλος, γράψτε το υπογεγραμμένο έγγραφο στο δίσκο. Μπορείτε να διατηρήσετε το αρχικό αρχείο αμετάβλητο και να δημιουργήσετε ένα νέο.

```csharp
// Save the signed PDF to a new file
pdfSigner.Save(@"C:\PDFs\signed_sha512.pdf");
```

Όταν ανοίξετε το `signed_sha512.pdf` στο Adobe Acrobat ή σε οποιονδήποτε προβολέα PDF που υποστηρίζει υπογραφές, θα δείτε ένα πράσινο σημάδι ελέγχου (ή το οπτικό στοιχείο που ορίσατε) και τις λεπτομέρειες του πιστοποιητικού.

## Πλήρες Παράδειγμα Λειτουργίας

Συνδυάζοντας όλα τα παραπάνω, εδώ είναι ένα ενιαίο, έτοιμο για αντιγραφή πρόγραμμα:

```csharp
using System;
using System.Drawing;
using Aspose.Pdf;
using Aspose.Pdf.Facades;
using Aspose.Pdf.Forms;

class Program
{
    static void Main()
    {
        // 1️⃣ Load the PDF you want to sign
        using var pdfDocument = new Document(@"C:\PDFs\toSign.pdf");

        // 2️⃣ Create the signature handler
        using var pdfSigner = new PdfFileSignature(pdfDocument);

        // 3️⃣ Build a PKCS#7 detached signature (SHA‑512)
        var pkcsSignature = new PKCS7Detached(
            @"C:\Certificates\mycert.pfx",
            "pwd",
            DigestHashAlgorithm.Sha512);

        // 4️⃣ Sign page 1 – you can change pageNumber or rectangle as needed
        pdfSigner.Sign(
            pageNumber: 1,
            isAppendMode: true,
            signatureRectangle: new Rectangle(100, 100, 200, 150),
            pkcsSignature);

        // 5️⃣ Save the result
        pdfSigner.Save(@"C:\PDFs\signed_sha512.pdf");

        Console.WriteLine("✅ PDF signed successfully!");
    }
}
```

Εκτελέστε το πρόγραμμα και θα δείτε το μήνυμα στην κονσόλα που επιβεβαιώνει την επιτυχία. Ανοίξτε το αρχείο εξόδου, ελέγξτε το τμήμα υπογραφών και θα δείτε τις πληροφορίες του πιστοποιητικού που παρείχατε.

## Πώς να Υπογράψετε PDF – Συχνές Παραλλαγές

### Υπογραφή Πολλών Σελίδων

Αν χρειάζεται να **προσθέσετε ψηφιακή υπογραφή** σε περισσότερες από μία σελίδες, καλέστε επανειλημμένα το `pdfSigner.Sign` με διαφορετικές τιμές `pageNumber`. Επειδή χρησιμοποιήσαμε `isAppendMode: true`, κάθε κλήση δημιουργεί μια νέα αυξητική ενημέρωση, διατηρώντας τις προηγούμενες υπογραφές.

### Χρήση Διαφορετικού Αλγορίθμου Κατακερματισμού

Ορισμένα παλαιά συστήματα καταλαβαίνουν μόνο SHA‑256. Αντικαταστήστε το `DigestHashAlgorithm.Sha512` με `DigestHashAlgorithm.Sha256` στον κατασκευαστή `PKCS7Detached`. Το υπόλοιπο του κώδικα παραμένει το ίδιο.

### Δημιουργία Αόρατης Υπογραφής

```csharp
pdfSigner.Sign(
    pageNumber: 1,
    isAppendMode: true,
    signatureRectangle: null,   // no visible appearance
    pkcsSignature);
```

Οι αόρατες υπογραφές είναι ιδανικές για αυτοματοποιημένες διαδικασίες παρτίδας όπου δεν απαιτείται οπτική ένδειξη.

### Επαλήθευση της Υπογραφής Προγραμματιστικά

```csharp
using Aspose.Pdf.Facades;

var verifier = new PdfFileSignature(@"C:\PDFs\signed_sha512.pdf");
bool isValid = verifier.VerifySignature(pageNumber: 1);
Console.WriteLine(isValid ? "Signature valid" : "Signature invalid");
```

### Διαχείριση PDF με Κωδικό Πρόσβασης

Αν το PDF προέλευσης είναι κρυπτογραφημένο, ανοίξτε το πρώτα με τον κωδικό πρόσβασης:

```csharp
var pdfDoc = new Document(@"C:\PDFs\protected.pdf", "pdfPassword");
```

Στη συνέχεια συνεχίστε με τα ίδια βήματα. Η υπογραφή θα εφαρμοστεί πάνω στο κρυπτογραφημένο περιεχόμενο.

## Επαγγελματικές Συμβουλές & Συνηθισμένα Πιθανά Σφάλματα

- **Ποτέ μην κωδικοποιείτε σκληρά κωδικούς** σε κώδικα παραγωγής. Χρησιμοποιήστε ασφαλή vaults ή μεταβλητές περιβάλλοντος.
- **Διατηρήστε το ιδιωτικό κλειδί του πιστοποιητικού προστατευμένο.** Αν το αρχείο `.pfx` εκτεθεί, οποιοσδήποτε μπορεί να πλαστογραφήσει έγγραφα.
- **Δοκιμάστε με διαφορετικούς προβολείς PDF.** Ορισμένοι παλαιότεροι αναγνώστες μπορεί να μην εμφανίζουν σωστά την υπογραφή αν λείπει το ρεύμα εμφάνισης.
- **Οι αυξητικές αποθηκεύσεις έχουν σημασία.** Αν ορίσετε `isAppendMode` σε `false`, οι υπάρχουσες υπογραφές θα ακυρωθούν επειδή το αρχείο ξαναγράφεται εντελώς.
- **Προσέξτε την περιστροφή σελίδας.** Οι συντεταγμένες του ορθογωνίου είναι σχετικές με την αρχική προσανατολισμό της σελίδας· οι περιστρεφόμενες σελίδες μπορεί να χρειάζονται προσαρμοσμένες συντεταγμένες.
