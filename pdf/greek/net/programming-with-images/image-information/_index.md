# Πληροφορίες εικόνας σε αρχείο PDF

## Εισαγωγή

Τα αρχεία PDF βρίσκονται παντού στις μέρες μας—σχεδόν κάθε επαγγελματικό και προσωπικό έγγραφο βρίσκει κάποια στιγμή τον δρόμο του σε αυτήν τη μορφή. Είτε πρόκειται για αναφορά, φυλλάδιο ή ηλεκτρονικό βιβλίο, η κατανόηση του τρόπου αλληλεπίδρασης με αυτά τα αρχεία μέσω προγραμματισμού προσφέρει μια πληθώρα δυνατοτήτων. Μια συνηθισμένη απαίτηση είναι η εξαγωγή πληροφοριών εικόνας από αρχεία PDF. Σε αυτόν τον οδηγό, θα εμβαθύνουμε στον τρόπο χρήσης της βιβλιοθήκης Aspose.PDF για .NET για την εξαγωγή κρίσιμων λεπτομερειών σχετικά με εικόνες ενσωματωμένες σε ένα έγγραφο PDF.

## Προαπαιτούμενα

Πριν μπούμε στα πιο απλά πράγματα της κωδικοποίησης, υπάρχουν μερικές προϋποθέσεις που θα πρέπει να έχετε στη διάθεσή σας:

1. Περιβάλλον Ανάπτυξης: Θα χρειαστείτε ένα περιβάλλον ανάπτυξης .NET. Αυτό θα μπορούσε να είναι το Visual Studio ή οποιοδήποτε άλλο IDE συμβατό με .NET.
2. Βιβλιοθήκη Aspose.PDF: Βεβαιωθείτε ότι έχετε πρόσβαση στη βιβλιοθήκη Aspose.PDF. Μπορείτε να την κατεβάσετε από το [Ιστότοπος Aspose](https://releases.aspose.com/pdf/net/). 
3. Βασικές γνώσεις C#: Η εξοικείωση με την C# και τις έννοιες του αντικειμενοστρεφούς προγραμματισμού θα σας βοηθήσει να παρακολουθήσετε το σεμινάριο χωρίς κόπο.
4. Έγγραφο PDF: Να έχετε πρόχειρο ένα δείγμα εγγράφου PDF που περιέχει εικόνες για να δοκιμάσετε τον κώδικά σας. 

## Εισαγωγή πακέτων

Για να ξεκινήσετε να χρησιμοποιείτε τη βιβλιοθήκη Aspose.PDF, θα χρειαστεί να εισαγάγετε τους απαραίτητους χώρους ονομάτων στο αρχείο C#. Ακολουθεί μια γρήγορη περίληψη:

```csharp
using System.IO;
using Aspose.Pdf;
using System;
```

Αυτοί οι χώροι ονομάτων θα σας παρέχουν πρόσβαση στις απαιτούμενες κλάσεις και μεθόδους για τον χειρισμό αρχείων PDF και την εξαγωγή δεδομένων εικόνας.

Τώρα που έχετε ρυθμίσει τα πάντα, ήρθε η ώρα να τα αναλύσουμε σε εύκολα διαχειρίσιμα βήματα. Θα γράψουμε ένα πρόγραμμα C# που φορτώνει ένα έγγραφο PDF, εξετάζει κάθε σελίδα και εξάγει τις διαστάσεις και την ανάλυση κάθε εικόνας στο έγγραφο.

## Βήμα 1: Αρχικοποίηση του εγγράφου

Σε αυτό το βήμα, θα αρχικοποιήσουμε το έγγραφο PDF χρησιμοποιώντας τη διαδρομή προς το αρχείο PDF σας. Θα πρέπει να αντικαταστήσετε `"YOUR DOCUMENT DIRECTORY"` με την πραγματική διαδρομή όπου βρίσκεται το αρχείο PDF σας.

```csharp
// Η διαδρομή προς τον κατάλογο εγγράφων.
string dataDir = "YOUR DOCUMENT DIRECTORY";
// Φόρτωση του αρχείου PDF προέλευσης
Document doc = new Document(dataDir + "ImageInformation.pdf");
```
Δημιουργούμε ένα `Document` αντικείμενο που φορτώνει το PDF από τον καθορισμένο κατάλογο. Αυτό θα μας επιτρέψει να εργαστούμε με τα περιεχόμενα του αρχείου.

## Βήμα 2: Ορισμός προεπιλεγμένης ανάλυσης και αρχικοποίηση δομών δεδομένων

Στη συνέχεια, ορίζουμε μια προεπιλεγμένη ανάλυση για τις εικόνες, η οποία είναι χρήσιμη για τους υπολογισμούς. Θα προετοιμάσουμε επίσης έναν πίνακα για να περιέχει ονόματα εικόνων και μια στοίβα για τη διαχείριση γραφικών καταστάσεων.

```csharp
// Ορίστε την προεπιλεγμένη ανάλυση για την εικόνα
int defaultResolution = 72;
System.Collections.Stack graphicsState = new System.Collections.Stack();
// Ορίστε αντικείμενο λίστας πινάκων που θα περιέχει ονόματα εικόνων
System.Collections.ArrayList imageNames = new System.Collections.ArrayList(doc.Pages[1].Resources.Images.Names);
```
Ο `defaultResolution` Η μεταβλητή μας βοηθά να υπολογίσουμε σωστά την ανάλυση των εικόνων. Η `graphicsState` Η στοίβα χρησιμεύει ως μέσο αποθήκευσης της τρέχουσας γραφικής κατάστασης του εγγράφου όταν συναντάμε τελεστές μετασχηματισμού.

## Βήμα 3: Επεξεργασία κάθε χειριστή στη σελίδα

Τώρα επαναλαμβάνουμε όλους τους τελεστές στην πρώτη σελίδα του εγγράφου. Εδώ είναι που γίνεται η πιο δύσκολη δουλειά. 

```csharp
foreach (Operator op in doc.Pages[1].Contents)
{
    // Χειριστές διεργασιών...
}
```
Κάθε τελεστής σε ένα αρχείο PDF είναι μια εντολή που υποδεικνύει στον renderer πώς να διαχειρίζεται γραφικά στοιχεία, συμπεριλαμβανομένων των εικόνων.

## Βήμα 4: Χειρισμός τελεστών GSave/GRestore

Μέσα στον βρόχο, θα χειριστούμε τις εντολές αποθήκευσης και επαναφοράς γραφικών για να παρακολουθούμε τις αλλαγές που γίνονται στην γραφική κατάσταση.

```csharp
if (opSaveState != null) 
{
    // Αποθήκευση προηγούμενης κατάστασης
    graphicsState.Push(((Matrix)graphicsState.Peek()).Clone());
} 
else if (opRestoreState != null) 
{
    // Επαναφορά προηγούμενης κατάστασης
    graphicsState.Pop();
}
```
`GSave` αποθηκεύει την τρέχουσα γραφική κατάσταση, ενώ `GRestore` Επαναφέρει την τελευταία αποθηκευμένη κατάσταση, επιτρέποντάς μας να επαναφέρουμε τυχόν μετασχηματισμούς κατά την επεξεργασία εικόνων.

## Βήμα 5: Διαχείριση πινάκων μετασχηματισμού

Στη συνέχεια, χειριζόμαστε τη συνένωση των μητρών μετασχηματισμού κατά την εφαρμογή μετασχηματισμών σε εικόνες.

```csharp
else if (opCtm != null) 
{
    Matrix cm = new Matrix(
        (float)opCtm.Matrix.A,
        (float)opCtm.Matrix.B,
        (float)opCtm.Matrix.C,
        (float)opCtm.Matrix.D,
        (float)opCtm.Matrix.E,
        (float)opCtm.Matrix.F);
    
    ((Matrix)graphicsState.Peek()).Multiply(cm);
    continue;
}
```
Όταν εφαρμόζεται ένας πίνακας μετασχηματισμού, τον πολλαπλασιάζουμε με τον τρέχοντα πίνακα που είναι αποθηκευμένος στην κατάσταση γραφικών, ώστε να μπορούμε να παρακολουθούμε οποιαδήποτε κλιμάκωση ή μετάφραση που εφαρμόζεται στην εικόνα.

## Βήμα 6: Εξαγωγή πληροφοριών εικόνας

Τέλος, επεξεργαζόμαστε τον τελεστή σχεδίασης για εικόνες και εξάγουμε τις απαραίτητες πληροφορίες, όπως διαστάσεις και αναλύσεις.

```csharp
else if (opDo != null) 
{
    // Χειριστής Do που σχεδιάζει αντικείμενα
    if (imageNames.Contains(opDo.Name)) 
    {
        Matrix lastCTM = (Matrix)graphicsState.Peek();
        XImage image = doc.Pages[1].Resources.Images[opDo.Name];
        double scaledWidth = Math.Sqrt(Math.Pow(lastCTM.Elements[0], 2) + Math.Pow(lastCTM.Elements[1], 2));
        double scaledHeight = Math.Sqrt(Math.Pow(lastCTM.Elements[2], 2) + Math.Pow(lastCTM.Elements[3], 2));
        double originalWidth = image.Width;
        double originalHeight = image.Height;
        
        double resHorizontal = originalWidth * defaultResolution / scaledWidth;
        double resVertical = originalHeight * defaultResolution / scaledHeight;
        
        // Εξαγωγή των πληροφοριών
        Console.Out.WriteLine(string.Format(dataDir + "image {0} ({1:.##}:{2:.##}): res {3:.##} x {4:.##}",
                         opDo.Name, scaledWidth, scaledHeight, resHorizontal, resVertical));
    }
}
```
Εδώ, ελέγχουμε αν ο χειριστής είναι υπεύθυνος για τη σχεδίαση μιας εικόνας. Εάν ναι, λαμβάνουμε το αντίστοιχο αντικείμενο XImage, υπολογίζουμε τις κλιμακωτές διαστάσεις και την ανάλυσή του και εκτυπώνουμε τις απαραίτητες πληροφορίες.
