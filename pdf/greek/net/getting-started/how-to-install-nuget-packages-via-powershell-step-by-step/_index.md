# πώς να εγκαταστήσετε πακέτα nuget μέσω PowerShell – βήμα προς βήμα

Έχετε αναρωτηθεί ποτέ **πώς να εγκαταστήσετε nuget** πακέτα χωρίς να ανοίξετε το Visual Studio; Δεν είστε μόνοι. Σε πολλά CI pipelines ή σε νέους υπολογιστές, η πιο γρήγορη λύση είναι να μπείτε στο PowerShell—κατά προτίμηση *run powershell as admin*—και να αφήσετε τον διαχειριστή πακέτων να κάνει τη δουλειά του.

Σε αυτό το tutorial θα περάσουμε από όλη τη διαδικασία: το άνοιγμα του σωστού console, τη λήψη μιας συγκεκριμένης έκδοσης μιας βιβλιοθήκης, και τελικά την επιβεβαίωση ότι το πακέτο πραγματικά εγκαταστάθηκε στο σύστημά σας. Στο τέλος θα μπορείτε να **list installed packages**, να γνωρίζετε την **how to verify package** ακεραιότητα, και να νιώθετε σίγουροι ότι το βήμα **verify installed package** πέτυχε κάθε φορά.

## Τι θα μάθετε

- Πώς να εκκινήσετε το PowerShell με τα σωστά προνόμια.  
- Η ακριβής σύνταξη της εντολής `Install-Package` για το NuGet.  
- Τρόποι για **list installed packages** και επιβεβαίωση αριθμών εκδόσεων.  
- Συνηθισμένα προβλήματα (έλλειψη δικαιωμάτων admin, ασυμφωνίες εκδόσεων) και πώς να τα αποφύγετε.  

Δεν απαιτείται προηγούμενη εμπειρία με το NuGet, μόνο ένας λειτουργικός υπολογιστής Windows και λίγη περιέργεια.

## Πώς να εγκαταστήσετε πακέτα NuGet χρησιμοποιώντας PowerShell

> **Pro tip:** Αν προσθέτετε συχνά τα ίδια πακέτα, σκεφτείτε να τα προσθέσετε σε ένα αρχείο script και να το εκτελέσετε με `-File`. Σας εξοικονομεί το να πληκτρολογείτε την ίδια γραμμή ξανά και ξανά.

### Βήμα 1: Ανοίξτε το PowerShell με τα απαραίτητα δικαιώματα

Το πρώτο πράγμα που πρέπει να κάνετε είναι **run powershell as admin**. Χωρίς ανυψωμένα δικαιώματα η cmdlet `Install-Package` μπορεί να αποτύχει σιωπηρά ή να ζητήσει επιβεβαίωση που δεν θέλετε να αντιμετωπίσετε.

1. Κάντε κλικ στο κουμπί Έναρξη.  
2. Πληκτρολογήστε **PowerShell**.  
3. Κάντε δεξί κλικ στο *Windows PowerShell* και επιλέξτε **Run as administrator**.  

Θα δείτε ένα παράθυρο UAC· κάντε κλικ στο **Yes**. Τώρα έχετε μια προνομιούχα συνεδρία έτοιμη για εγκατάσταση πακέτων.

> *Γιατί admin?*  
> NuGet γράφει αρχεία στο φάκελο παγκόσμιων πακέτων (`C:\Program Files\PackageManagement\NuGet\Packages` εξ ορισμού). Η τοποθεσία αυτή είναι προστατευμένη, οπότε μόνο μια ανυψωμένη διαδικασία μπορεί να γράψει εκεί.

### Βήμα 2: Εγκαταστήστε το επιθυμητό πακέτο NuGet και την έκδοση

Με το console ανοιχτό, η βασική εντολή είναι απλή:

```powershell
# Install the Aspose.PDF library, version 25.3
Install-Package Aspose.PDF -Version 25.3
```

- `Install-Package` είναι το PowerShell wrapper γύρω από τον client του NuGet.  
- `-Version` κλειδώνει την ακριβή έκδοση που χρειάζεστε, αποτρέποντας τυχαίες αναβαθμίσεις.  

Αν παραλείψετε το `-Version`, το PowerShell θα κατεβάσει την πιο πρόσφατη σταθερή έκδοση—μερικές φορές είναι εντάξει, άλλες φορές θέλετε την ακριβή έκδοση που δοκιμάσατε.

#### Τι συμβαίνει παρασκηνιακά;

Το PowerShell επικοινωνεί με την ρυθμισμένη πηγή πακέτων (εξ ορισμού `https://www.nuget.org/api/v2`) και κατεβάζει το αρχείο `.nupkg`. Στη συνέχεια εξάγει τα DLLs στον φάκελο παγκόσμιων πακέτων και καταχωρεί το πακέτο στον τοπικό πάροχο πακέτων. Η όλη διαδικασία συνήθως ολοκληρώνεται σε λίγα δευτερόλεπτα εκτός αν έχετε αργό δίκτυο.

### Βήμα 3: Επαληθεύστε ότι το πακέτο εγκαταστάθηκε επιτυχώς

Τώρα που το πακέτο είναι στον δίσκο, πιθανότατα θα ρωτήσετε, **«Πώς επαληθεύω το πακέτο;»** Η απάντηση βρίσκεται σε ένα απλό ερώτημα:

```powershell
# List all installed NuGet packages
Get-Package -Name Aspose.PDF
```

Η εκτέλεση αυτού επιστρέφει κάτι σαν:

```
Name        Version   Source
----        -------   ------
Aspose.PDF  25.3      nuget.org
```

Αυτή η έξοδος επιβεβαιώνει δύο πράγματα:

1. Το πακέτο **Aspose.PDF** είναι παρόν.  
2. Η έκδοσή του ταιριάζει με αυτή που ζητήσατε, ικανοποιώντας την απαίτηση **verify installed package**.

Αν θέλετε να δείτε *κάθε* πακέτο στον υπολογιστή, αφαιρέστε το φίλτρο `-Name`:

```powershell
Get-Package | Where-Object {$_.ProviderName -eq 'NuGet'}
```

Αυτή η προβολή **list installed packages** είναι χρήσιμη για ελέγχους ή όταν χρειάζεται να καθαρίσετε παλιές βιβλιοθήκες.

### Βήμα 4: Προαιρετικό – διαχείριση ειδικών περιπτώσεων

#### α) Πακέτο δεν βρέθηκε ή ασυμφωνία έκδοσης

Αν το PowerShell απαντήσει με *«Package not found»* ή *«Version not available»*, ελέγξτε ξανά την ορθογραφία και τον αριθμό έκδοσης. Το NuGet δεν διακρίνει πεζά‑κεφαλαία, αλλά ένα περιττό κενό θα σπάσει την εντολή.

```powershell
# Search the NuGet feed for available versions
Find-Package Aspose.PDF -AllVersions
```

#### β) Εκτέλεση χωρίς δικαιώματα admin

Αν ξεχάσετε να **run powershell as admin**, η cmdlet θα ρίξει σφάλμα δικαιωμάτων. Η λύση είναι απλώς να κλείσετε το παράθυρο και να το ανοίξετε ξανά με ανυψωμένα δικαιώματα—χωρίς ανάγκη επανεγκατάστασης.

#### γ) Χρήση προσαρμοσμένης πηγής

Σε εταιρικά περιβάλλοντα μπορεί να έχετε ένα εσωτερικό NuGet feed:

```powershell
Install-Package MyCompany.Logging -Source https://nuget.mycompany.local/api/v2
```

Το βήμα επαλήθευσης παραμένει το ίδιο· απλώς θυμηθείτε να συμπεριλάβετε το `-Source` όταν εγκαθιστάτε.

## Πίνακας γρήγορης αναφοράς

| Δράση                              | PowerShell command                                          | Γιατί είναι σημαντικό |
|-------------------------------------|-------------------------------------------------------------|------------------------|
| Άνοιγμα ανυψωμένου console               | *Run PowerShell as Administrator*                           | Απαιτείται για παγκόσμια εγκατάσταση |
| Εγκατάσταση συγκεκριμένης έκδοσης          | `Install-Package <pkg> -Version <x.y.z>`                    | Εγγυάται αναπαραγώγιμες κατασκευές |
| Λίστα ενός μόνο πακέτου               | `Get-Package -Name <pkg>`                                    | Επιβεβαιώνει **how to verify package** |
| Λίστα όλων των πακέτων NuGet             | `Get-Package | Where-Object {$_.ProviderName -eq 'NuGet'}`| Χρήσιμο για **list installed packages** |
| Αναζήτηση διαθέσιμων εκδόσεων           | `Find-Package <pkg> -AllVersions`                           | Βοηθά όταν η έκδοση είναι άγνωστη |

## Συμπέρασμα

Καλύψαμε **how to install nuget** πακέτα χρησιμοποιώντας PowerShell από την αρχή μέχρι το τέλος—ανοίγοντας το console **run powershell as admin**, κατεβάζοντας μια συγκεκριμένη έκδοση, και τελικά **list installed packages** για **verify installed package**. Με αυτές τις εντολές στο toolbox σας μπορείτε να αυτοματοποιήσετε τη διαχείριση βιβλιοθηκών σε οποιονδήποτε υπολογιστή Windows, είτε γράφετε script για CI pipeline είτε απλώς διορθώνετε ένα ελλειπτικό DLL στο dev box σας.

Επόμενα βήματα; Δοκιμάστε να προσθέσετε πολλαπλά πακέτα σε ένα μόνο script, εξερευνήστε την παράμετρο `-Scope` για τοπική εγκατάσταση σε έργο, ή συνδυάστε αυτές τις εντολές με `Invoke-Expression` για να δημιουργήσετε έναν ελαφρύ εγκαταστάτη για την ομάδα σας. Και αν αντιμετωπίσετε πρόβλημα, θυμηθείτε το βήμα **how to verify package**—η προβολή της έκδοσης στο `Get-Package` είναι συχνά ο πιο γρήγορος τρόπος να εντοπίσετε το πρόβλημα.

Καλή διασκέδαση με το PowerShell! 🚀
