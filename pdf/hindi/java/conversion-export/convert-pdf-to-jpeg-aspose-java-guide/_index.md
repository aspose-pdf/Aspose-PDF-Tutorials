## सुविधा 1: सभी PDF पृष्ठों को JPEG छवियों में बदलें

### “convert all pdf pages” का क्या अर्थ है?
यह सुविधा PDF दस्तावेज़ के प्रत्येक पृष्ठ को क्रमशः पढ़ती है और प्रत्येक पृष्ठ को एक अलग JPEG फ़ाइल के रूप में सहेजती है। यह **batch convert pdf jpeg** परिदृश्यों जैसे इमेज आर्काइव बनाना या वेब गैलरी के लिए थंबनेल तैयार करने में आदर्श है।

### चरण‑दर‑चरण कार्यान्वयन

**चरण 1: PDF दस्तावेज़ लोड करें**  
हम एक `Document` ऑब्जेक्ट बनाते हैं जो स्रोत PDF का प्रतिनिधित्व करता है।

```java
Document pdfDocument = new Document(dataDir + "/input.pdf");
```

**चरण 2: प्रत्येक पृष्ठ पर इटररेट करें**  
एक साधारण `for` लूप हर पृष्ठ इंडेक्स को पार करता है।

```java
for (int pageCount = 1; pageCount <= pdfDocument.getPages().size(); pageCount++) {
    // Processing code here
}
```

**चरण 3: आउटपुट और रूपांतरण पैरामीटर सेट करें**  
`JpegDevice` को 300 DPI और अधिकतम गुणवत्ता (100) के साथ कॉन्फ़िगर करें।

```java
Resolution resolution = new Resolution(300);
JpegDevice jpegDevice = new JpegDevice(resolution, 100);
```

**चरण 4: प्रत्येक पृष्ठ को JPEG में बदलें**  
लूप के भीतर, हम प्रत्येक पृष्ठ को JPEG फ़ाइल में लिखते हैं, और स्ट्रीम को सही ढंग से बंद करने के लिए `try‑with‑resources` ब्लॉक का उपयोग करते हैं।

```java
try (OutputStream imageStream = new FileOutputStream(outputDir + "/Converted_Image" + pageCount + ".jpg")) {
    jpegDevice.process(pdfDocument.getPages().get_Item(pageCount), imageStream);
}
```

#### सामान्य गलतियाँ और टिप्स
- **Path correctness:** `dataDir` और `outputDir` मानों को दोबारा जांचें; रिलेटिव पाथ अक्सर गलत हो जाते हैं।  
- **File permissions:** सुनिश्चित करें कि जावा प्रक्रिया को आउटपुट फ़ोल्डर में लिखने की अनुमति है।  
- **Memory usage:** बहुत बड़े PDFs के लिए पृष्ठों को छोटे बैच में प्रोसेस करने या JVM हीप आकार बढ़ाने पर विचार करें।

### “convert single pdf page” कब उपयोगी है?

### चरण‑दर‑चरण कार्यान्वयन

**चरण 1: PDF दस्तावेज़ लोड करें** (पहले जैसा)

```java
Document pdfDocument = new Document(dataDir + "/input.pdf");
```

**चरण 2: आउटपुट स्ट्रीम और रूपांतरण पैरामीटर सेट करें**  
उसी `Resolution` और `JpegDevice` कॉन्फ़िगरेशन को पुनः उपयोग करें।

```java
Resolution resolution = new Resolution(300);
JpegDevice jpegDevice = new JpegDevice(resolution, 100);
```

**चरण 3: निर्दिष्ट पृष्ठ को JPEG में बदलें**  
यहाँ हम केवल पहला पृष्ठ बदलते हैं (विभिन्न पृष्ठ के लिए इंडेक्स बदलें)।

```java
try (OutputStream imageStream = new FileOutputStream(outputDir + "/Converted_Image.jpg")) {
    jpegDevice.process(pdfDocument.getPages().get_Item(1), imageStream);
}
```

#### त्वरित जाँच‑सूची
- पृष्ठ संख्या मौजूद है (`pdfDocument.getPages().size()`) यह सत्यापित करें।  
- यदि पृष्ठ इंडेक्स सीमा से बाहर है तो `IndexOutOfBoundsException` को संभालें।  
- संसाधन लीक से बचने के लिए वही `try‑with‑resources` पैटर्न उपयोग करें।

A: हाँ। रूपांतरण से पहले पासवर्ड पैरामीटर (`new Document(path, password)`) के साथ दस्तावेज़ लोड करें।

**Q: रूपांतरण वेक्टर ग्राफ़िक्स को कैसे संभालता है?**  
A: वेक्टर तत्व निर्दिष्ट DPI पर रास्टराइज़ होते हैं, जिससे परिणामी JPEG में दृश्य शुद्धता बनी रहती है।

**Q: यदि मुझे पारदर्शी बैकग्राउंड चाहिए तो क्या करें?**  
A: JPEG पारदर्शिता को सपोर्ट नहीं करता; यदि आपको अल्फा चैनल चाहिए तो PNG या TIFF पर स्विच करें।

**Q: क्या ट्रायल संस्करण वॉटरमार्क जोड़ता है?**  
A: मुफ्त ट्रायल प्रत्येक उत्पन्न छवि के पहले पृष्ठ पर एक छोटा वॉटरमार्क जोड़ता है। लाइसेंस्ड संस्करण सभी वॉटरमार्क हटाता है।
